| 단가 | 10원 |

| 1일투여횟수 | 2회 |
| 총투여일수 | 3일 |
| 시간가산 | 없음 |

### 단계별 계산

#### Step 2: 약품금액

```
소모량 = 0.5 * 2 * 3 = 3.0
약품금액 = (int)(3.0 * 10 + 0.5) = (int)(30.5) = 30원
```

> **검증 포인트:** `30.5`에 사사오입 → 31이 아닌 `(int)(30.5) = 30`이 되는지? C#의 `(int)` 캐스트는 소수부 절사(truncation)이다. 그러나 `(int)(x + 0.5m)` 패턴에서 `x=30.0`이면 `(int)(30.5) = 30`(C# int 캐스트는 truncate). **단, `Math.Round(30.0, MidpointRounding.AwayFromZero) = 30`이므로 결과 동일.** 실제로 약품금액이 정확히 정수인 경우 +0.5 후 절사해도 원래값과 같다.

```
약품금액 = 30원 (정확히 정수이므로 반올림 영향 없음)
```

#### Step 3: 조제료

| Z코드 | 단가(원) | 금액(원) |
|-------|---------|---------|
| Z1000 | 790 | 790 |
| Z2000 | 1,720 | 1,720 |
| Z3000 | 1,150 | 1,150 |
| Z4103 | 2,680 | 2,680 |
| Z5000 | 680 | 680 |
| **합계** | | **7,020** |

#### Step 5~7: 총액/부담/청구

```
총액1 = trunc10(30 + 7020) = trunc10(7050) = 7,050원
본인부담금 = trunc100(7050 * 0.30) = trunc100(2115) = 2,100원
청구액 = 7050 - 2100 = 4,950원

## 5. 시나리오 S03: 다중 가산 (6세미만 + 야간)

### 입력

| 항목 | 값 |
|------|-----|
| 환자나이 | 3세 |
| 보험코드 | C10 |
| 약품 | 내복약 1종 (급여) |
| 단가 | 500원 |
| 1회투약량 | 1정 |
| 1일투여횟수 | 3회 |
| 총투여일수 | 7일 |
| **야간여부** | **TRUE** |
| 산제 | N |

### 가산 결정 로직

```
가루약가산? → N → 다음 확인
야간? → Y → 야간 가산 적용
6세미만? → Y → 6세미만+야간 복합코드 적용 (Z2000610)
소아가산? → 야간과 6세미만이 함께 적용되면 별도 소아가산 없음
          (6세미만 코드 자체에 소아가산이 포함됨)
```

> **핵심 규칙:** 6세미만 + 야간 → `Z2000610`(기본조제료), `Z3000010`(복약지도료-야간), `Z4107010`(내복약조제료-야간) 적용. 기본조제료만 6세미만+야간 복합코드를 사용하고, 복약지도료/내복약조제료는 야간 코드를 사용한다.

### 단계별 계산

#### Step 2: 약품금액

```
약품금액 = (int)(1 * 3 * 7 * 500 + 0.5) = 10,500원
```

#### Step 3: 조제료

| Z코드 | 명칭 | 단가(원) | 금액(원) |
|-------|------|---------|---------|
| Z1000 | 약국관리료 | 790 | 790 |
| Z2000610 | 기본조제료(6세미만+야간) | 3,150 | 3,150 |
| Z3000010 | 복약지도료(야간) | 1,500 | 1,500 |
| Z4107010 | 내복약조제료(7일+야간) | 5,620 | 5,620 |
| Z5000 | 의약품관리료 | 680 | 680 |
| **합계** | | | **11,740** |

#### Step 5~7: 총액/부담/청구

```
총액1 = trunc10(10500 + 11740) = trunc10(22240) = 22,240원
```

**6세미만 본인부담률:**

```
C10 SixAgeRate = 50%
성인 부담률 30% × (100% - 50%) = 30% × 50% = 15%? 
```

> **주의: SixAgeRate의 해석**
> CH05/약국_약제비_계산_로직.md에 따르면 6세미만은 "성인 부담의 70%" = 30% × 70% = **21%** (시행령 별표2 제3호 다목 3)).
> 그런데 EDB Mock의 SixAgeRate=50은 "경감비율 50%"를 의미한다. 즉 성인 부담률의 (100% - 50%) = 50%를 부담한다는 뜻이다.
>
> **로직 문서(법령 기준):** 6세미만 = 30% × 70% = **21%** → `trunc100(22240 * 0.21)`
> **EDB Mock(SixAgeRate=50):** 6세미만 = 30% × (1 - 0.50) = **15%** → `trunc100(22240 * 0.15)`
>
> 이 불일치는 EDB가 약국이 아닌 의료기관 공통 설정(의원급 6세미만 50% 경감)을 사용하기 때문이다.
> **법령 기준(약국)은 21%가 정확하다.** 테스트 시 아래 두 값 모두 계산한다.

**법령 기준 (권장):**

```
본인부담금 = trunc100(22240 * 0.21) = trunc100(4670.4) = 4,600원
청구액 = 22240 - 4600 = 17,640원
```

**EDB Mock SixAgeRate 적용 시 (참고):**

```
본인부담금 = trunc100(22240 * 0.15) = trunc100(3336) = 3,300원
청구액 = 22240 - 3300 = 18,940원
```

## 6. 시나리오 S04: 혼합 보험 (급여 + 비급여 + 전액본인부담)

### 입력

**약품 3종:**

| # | 복용구분 | 급여구분 | 단가 | 1회투약량 | 1일횟수 | 총일수 |

| 1 | 내복약 | 급여(01항) | 500원 | 1정 | 3회 | 7일 |

### 단계별 계산

#### Step 2: 약품금액

```
약품1(급여) = (int)(1*3*7*500 + 0.5) = 10,500원 → 01항
약품2(비급여) = (int)(1*2*7*300 + 0.5) = 4,200원 → W항
