# CH02. 조제료 코드체계 (Z코드 완전 레퍼런스)

> **목적:** 다른 AI가 이 문서만 보고 Z코드를 생성하고 수가를 조회할 수 있도록, 약국 조제료에 사용되는 Z코드 전체를 코드 구조 / 마스터 목록 / 접미사(산정코드) 체계 / 투약일수별 코드 생성 로직까지 하나로 정리한다.
>
> **소스:** 비즈팜(BizPharm) / 공단(NHIS) / 유팜(UPharm) / EDB 4개 소스 제출물 + `docs/약국_약제비_계산_분석.md` + `docs/약국_약제비_계산_로직.md`

---

## 1. 개요

약국 약제비 청구 시 **02항 01목(조제료 등)** 에 기재되는 행위 수가는 모두 **Z코드**로 식별된다.

```
약제비 = 약가(01항) + 조제료(02항)
조제료 = 기본조제료 + 약품조제료

기본조제료 = 약국관리료(Z1000)
           + 기본조제기술료(Z2000)
           + 복약지도료(Z3000)
           + 의약품관리료(Z5xxx)
           + [야간조제관리료(Z7001)]
           + [비대면조제관리료(ZCxxx)]

약품조제료 = 내복조제료(Z4xxx) + 외용조제료(Z4120/Z4220)
           + 주사조제료(Z4130) + 내외용동시(Z4121/Z4221)
           + [가루약가산(Z4010/Z4020)]
```

Z코드는 크게 **기본코드 5자리**와 **산정코드(접미사) 3~4자리**가 결합하여 최종 청구코드를 형성한다. 접미사가 모두 빈 값이면 기본코드만 사용한다.

## 2. Z코드 기본 구조

### 2-1. 코드 구조 (기본코드 + 산정코드)

```
[최종 청구코드] = [기본코드 5자리] + [산정코드(접미사) 3자리]

예시:
  Z1000         → 약국관리료 (접미사 없음)
  Z2000 + 010   → 조제기본료 야간 (Z2000010)
  Z4107 + 610   → 7일 내복약조제료 + 6세미만 + 야간 (Z4107610)
  Z4326 + 030   → 26~30일 내복약조제료 + 토요가산 (Z4326030)
```

### 2-2. 기본코드 대분류 (공단 체계)

공단에서 사용하는 Z코드 대분류 범위는 다음과 같다.

| 대분류 | 범위 | 항목 | 비고 |
|--------|------|------|------|
| Z10 | Z1000~Z1xxx | 약국관리료 | 방문당 1회 |
| Z20 | Z2000~Z2xxx | 조제기본료(기본조제기술료) | 방문당 1회 |
| Z30 | Z3000~Z3xxx | 복약지도료 | 방문당 1회 |
| Z40 | Z40xx | 내복약 조제료 (기타) | 가루약(Z4010/Z4020), 팩단위(Z4100/Z4201) |
| Z41 | Z4101~Z4115 | 처방전 내복약 조제료 (1~15일) | 1일 단위 개별코드 |
| Z42 | Z4200~Z4221 | 직접조제 (처방전 없음) | 내복/외용 |
| Z43 | Z4316~Z4391 | 내복약 조제료 (16일 이상 구간) | 구간제 |
| Z50 | Z5000~Z5011 | 의약품관리료 | 방문당, 마약/병팩 변형 |
| Z70 | Z7001 | 야간조제관리료 | 공공야간약국 한정 |

### 2-3. 산정코드(접미사) 3자리 구조

```
접미사 = text(1자리) + text2(1자리) + text3(1자리)

text  (의미I/연령) : ""=일반, "6"=6세미만, "1"=가루약(Z41/Z43에서만)
text2 (의미II/시간): ""=일반, "1"=야간, "2"=심야+6세미만, "3"=토요, "4"=심야+6세미만(Z20), "5"=공휴
text3 (차등)       : ""=해당없음, "0"=차등수가 해당, "1"=차등수가 비해당
```

> **핵심:** 접미사의 각 자릿수가 빈 문자열("")이면 해당 자리를 생략한다. 결과적으로 접미사가 "010"이면 야간, "030"이면 토요, "610"이면 6세미만+야간 등으로 조합된다.

## 4. 접미사(산정코드) 체계 상세

### 4-1. 접미사 3자리 완전 매핑

EDB 소스에서 확인된 접미사 조합의 **완전 매핑**이다. `text` + `text2` + `text3` 3개 필드를 연결하여 최종 접미사를 생성한다.

#### text (의미I / 연령 / 가루약)

| 값 | 의미 | 적용 대상 Z코드 |
|----|------|-----------------|
| "" (빈값) | 일반 | 모든 Z코드 |
| "6" | 6세미만 | Z2000 (조제기본료) |
| "1" | 가루약 조제투약 가산 | Z41xx, Z43xx (처방전 내복약 조제료) |
| "1" | 차등수가 제외 | Z1000 (약국관리료, text 위치에서 의미I로 사용) |

> **주의:** text="6"은 Z2000에서만 사용된다. Z41xx/Z43xx에서 6세미만은 text가 아닌 다른 경로(소아가산점수)로 처리된다. text="1"은 Z1000에서는 차등수가 제외, Z41xx/Z43xx에서는 가루약 가산이라는 전혀 다른 의미를 가진다.

#### text2 (의미II / 시간대) — 공단 고시 기준 분리 매핑

| 값 | 의미 | 적용 Z코드 (대상) | 가산률 |
|----|------|---|---|
| "" (빈값) | 일반 | 모든 Z코드 | - |
| "1" | 야간 (평일 18시~익일 09시) | Z2000/Z3000/Z41xx/Z43xx/Z4120/Z4121/Z4130 | 30% |
| **"2"** | **심야 (6세 미만 전용, 20시~익일 07시) — Z40 조제료** | Z41xx, Z43xx, Z4120, Z4121, Z4130 | **100% 가산** |
| "3" | 토요 (09시~13시) | 모든 Z코드 | 30% 별도 산정 |
| **"4"** | **심야 (6세 미만 전용, 20시~익일 07시) — Z20/Z30 조제기본료·복약지도료** | Z2000, Z3000 | **200% 가산** |
| "5" | 공휴일 | 모든 Z코드 | 30% |

> **검증 노트 (CH02 점검 2026-04-09, 공단 고시 PDF 직접 인용):**
>
> **출처:** 보건복지부 고시 제2025-186호 「건강보험요양급여비용의 내역」 페이지 687 산정지침 (9):
> > "위 「(8)」의 규정에도 불구하고, **6세 미만의 소아에 대하여 20시~익일 07시에 조제투약하는 경우** 다음과 같이 가산한다.
> > **(가) 조제기본료(약-2), 복약지도료(약-3) 소정점수의 200%(산정코드 두 번째 자리에 4로 기재)**
> > **(나) 조제료(약-4) 소정점수의 100%(산정코드 두 번째 자리에 2로 기재)**"
>
> **핵심 사실:**
> 1. **text2="2" vs "4"는 정말 다른 코드.** Z20/Z30(조제기본료/복약지도료)와 Z40(조제료)에서 6세 심야 가산 코드가 분리됨.
> 2. **심야 가산은 6세 미만 전용.** 일반 성인은 18-9시 야간 가산(text2="1", 30%)만 적용됨.
> 3. **사양 정정 이력:**
>    - 사양 초판: "Z20=4, Z41/Z43=2 분리" (정답)
>    - CH02 점검 1차 (2026-04-08): EDB 분석가 보고만 보고 "통일 매핑"으로 잘못 정정
>    - CH04 점검 1차 (2026-04-08): "6세 결합은 64 별도 패턴" 부분 인정
>    - **CH02 점검 2차 (2026-04-09):** 공단 고시 원문 직접 확인 → **사양 초판이 맞았음. 다시 분리 매핑으로 되돌림.**
> 4. EDB 코드는 두 가산률 필드(`소아야간가산률` Z40용 / `소아야간가산률_기본` Z20·Z30용)로 분리 관리. 유팜 `Base조제료Logic.cs:128`에서 `EqualsAny(기본조제기술료, 복약지도료)` 분기로 확인.
>

### 5-3. 의약품관리료 Z5xxx 처리 — **가짜 의사코드 폐기됨 (CH02 점검 2026-04-08)**

> **⚠ 사양 정정:** 사양 초판은 본 항목에 `GetMedMgmtSugaCD()` 의사코드(Z5101~Z5115 + Z5316~Z5391 25구간)를 명시했으나, **§3-4와 함께 통째로 폐기**됨. 4팀 검증 결과 EDB·유팜·비즈팜·공단 어디에도 Z5xxx 25구간 일수별 코드가 존재하지 않음. 사양 작성자가 내복약 `GetInJojeSugaCD()`(§5-2) 패턴을 의약품관리료에 무근거로 복사한 오류.

**실제 의약품관리료 코드 결정 (EDB 기준):**

```
function GetMedMgmtSugaCD(약품정보) -> string:
    if 모든 약품이 팩단위:
        return "Z5011"   // 병팩, 투약일수 1일 고정
    if 외용약만 처방:
        return "Z5010"   // 외용 단독

    return "Z5000"        // 일반
```

→ **단일 코드 + 단가 분기**. 일수별 별도 코드 없음. 일수 효과는 단가 마스터(`MockCostCalcRepository.cs`의 Z5xxx 단가 행)에서 시점/조건별로 처리.

상세 처리는 **CH03 §3-5** 참조.

### 5-4. 투약일수 산정 규칙

```
[조제일수인정 분기]
조제일수인정 == 1 → 조제매출.조제일수 (기본값) 그대로 사용
조제일수인정 != 1 → 약품별 최대값 계산:

[급여 조제일수]
최대내복조제일수 = Max(
  약품ID>0 AND 항목구분="0301" AND 팩수량<=1 AND !조제료산정여부
  AND (보험 OR 전액본인부담 OR 보훈전액본인부담 OR 코로나19치료제)
  인 약품의 투약일수
)

[급여 가루약 조제일수]
최대가루약조제일수 = Max(
  약품ID>0 AND 항목구분="0301" AND 팩수량<=1 AND 가루약=TRUE
  AND (보험 OR 전액본인부담 OR 보훈전액본인부담)
  AND !코로나19치료제
  인 약품의 투약일수
)

[비급여 조제일수]
비급여 약품 또는 팩수량>1인 약품 → 1일로 강제

[투약일수 → 점수 구간]
1~15일  → 해당 일수의 개별코드 점수 적용
16일 이상 → 해당 구간의 일괄 점수 적용
```

> **유팜 소스 확인:** 유팜은 내복 조제료를 2구간(1~15일 / 16일+)으로 단순화하여 DB에서 관리한다. ST조제료 테이블에서 `시작일`/`종료일` 필드로 구간을 구분하며, `Get기준금액()` 함수가 일별 정확 매칭 또는 구간 매칭을 수행한다.

## 6. 토요가산 코드 ("030" 접미사)

### 6-1. 토요가산 접미사 패턴

```

예시:
  Z2000030  → 조제기본료 토요가산
  Z3000030  → 복약지도료 토요가산
  Z4107030  → 7일분 내복약조제료 토요가산
  Z4326030  → 26~30일분 내복약조제료 토요가산
```

### 6-2. 토요가산 계산 방식

토요 가산은 기본금액과 **분리 계산** 후 합산한다. 이것은 다른 가산(야간/공휴)이 기본점수에 률을 곱하는 것과 다르다.

```
토요가산금액 = Math.Round(기본점수 × 토요일가산률, 2)
기본점수     = Math.Round(기본점수, 2)

최종금액 = (기준금액 × 기본점수).사사오입 + (기준금액 × 토요가산금액).사사오입
```

가루약 가산은 **내복약 조제료에만** 적용된다. 외용약/주사 조제료에는 미적용.

## 8. 처방조제 vs 직접조제 코드 비교표

| 구분 | 처방조제 | 직접조제 | 비고 |
|------|----------|----------|------|
| **점수 체계** | P점수 | D점수 | ST조제료 테이블에서 별도 관리 |
| **가산률** | P공휴가산률, P소아가산점수 | D공휴가산률, D소아가산점수 | |
| **내복약 단독** | Z4101~Z4115, Z4316~Z4391 | Z4200 | 직접조제는 단일코드 × 투약일수 |
| **내복약 가루약** | Z4010 | Z4020 | |
| **내복약 팩단위** | Z4100 | Z4201 | |
| **외용약 단독** | Z4120 | Z4220 | |
| **내외용 동시** | Z4121 | Z4221 | |
| **주사 단독** | Z4130 | (미배정) | 직접조제 주사는 Z코드 없음 |
| **야간조제관리료** | Z7001 (달빛어린이) | 미적용 | 처방조제 전용 |
| **접미사** | 전체 사용 | 사용 안 함 | 직접조제는 접미사/산정코드 없음 |
| **내복 금액 계산** | 코드별 고정 금액 | (기준금액 × D점수) × 투약일수 | 직접조제는 1일 점수 × 일수 |
| **의약품관리료** | Z5000/Z5001/Z5011 | Z5000 (일반만) | 직접조제는 마약/병팩 분기 없음 |

### 8-1. 직접조제 내복약 금액 계산 차이

```
처방조제 (Z4107):
  금액 = (기준금액 × Z4107의_P점수).사사오입  → 단일 금액

직접조제 (Z4200):
  1일금액 = (기준금액 × Z4200의_D점수).사사오입
  금액 = 1일금액 × 투약일수
  → Get조제료Item() 오버라이드에서 내복단독_직접일 때만 투약일수 배수
```

## 10. 주의사항

### 10-1. Z코드 생성 시 절대 금지 사항

1. **존재하지 않는 Z코드 생성 금지**: Z4319, Z4317 같은 코드는 존재하지 않음. 16일 이상은 반드시 구간 코드(Z4316, Z4321 등)만 사용
2. **접미사 규칙 혼용 금지**: Z2000의 접미사 규칙과 Z4107의 접미사 규칙이 다름. Z코드별로 허용된 접미사만 사용
3. **직접조제에 접미사 부여 금지**: Z4200~Z4221은 접미사 없음

### 10-2. 코드 결정 시 확인 순서

```
1. 처방조제 vs 직접조제 → 코드 계열 결정 (Z41/Z43 vs Z42)
2. 약품 조합 확인 → 내복/외용/주사/동시 코드 결정
3. 팩단위 판정 → Z4100/Z4201 분기
4. 가루약 판정 → Z4010/Z4020 또는 가산률 분기 (2023.11.01 전후)
5. 투약일수 → 25구간 코드 매핑 (GetInJojeSugaCD)
6. 시간대/연령 → 접미사 생성 (text+text2+text3)
7. 의약품관리료 → Z5000/Z5001/Z5011 또는 Z5xxx 구간코드
8. 특수 조제료 → ZC/ZH/ZE 해당 여부
```

### 10-3. 약품 조합별 Z코드 자동 결정 (Quick Reference)

```
[처방조제]
내복만           → Z41xx/Z43xx (또는 Z4100 팩, Z4010 가루)
외용만           → Z4120
내복+외용        → Z41xx/Z43xx + Z4121
주사만(자가투여) → Z4130 (기본조제료 정상)
주사만(비자가)   → 조제료 미산정 (약가만)
내복+주사        → Z41xx/Z43xx + Z4130
외용+주사        → Z4120 + Z4130
내복+외용+주사   → Z41xx/Z43xx + Z4121 + Z4130

[직접조제]
내복만           → Z4200 (또는 Z4201 팩, Z4020 가루)
외용만           → Z4220
내복+외용        → Z4200 + Z4221
주사             → 미배정 (직접조제 주사 없음)
```

### 10-4. 의약품관리료 코드 결정 분기

```
[처방조제]
급여 마약약품 포함        → Z5001 (마약)
모든 약품이 팩단위+비향정 → Z5011 (병팩, 투약일수 1일 고정)
그 외                     → Z5000 (일반)

[직접조제]
일률적으로               → Z5000 (일반만)

[비급여 마약류 (2019.01.01~)]
기본 미산정, 환경설정 "비급여마약류관리료적용"=TRUE면 산정
비급여 마약관리료 차액 = Z5001금액 - Z5000금액 (양수만)
```

### 10-5. 가산 적용 우선순위

```
1순위: 가루약 가산 → 다른 모든 가산 배제
2순위: 야간/공휴 가산
3순위: 소아심야 가산
4순위: 토요 가산
추가: 소아 가산 (2~4순위와 중복 적용 가능, 단 가루약 시 미적용)
비대면 조제(ZCxxx) 시: 모든 가산 미적용
```

### 10-6. 비대면 조제 코드 결정 우선순위

```
① 공휴여부=TRUE              → ZC004 (시간대 무관, 최우선)
② 야간여부=TRUE AND 심야시간  → ZC003 (20시~07시)
③ 야간여부=TRUE AND 위 외    → ZC002 (야간)
④ 그 외                      → ZC001 (일반)
```

### 10-7. 수가 조회 필수 정보

Z코드로 수가(금액)를 조회하려면 다음이 필요하다:

| 필요 정보 | 용도 |
|-----------|------|
| Z코드 (기본코드 + 접미사) | 수가 행 식별 |
| 조제일(내방일) | 적용일별 수가표 선택 (ST조제료적용일) |
| P/D 구분 | 처방조제=P점수, 직접조제=D점수 |
| 투약일수 | 구간 코드 결정 + 직접조제 배수 계산 |

```
수가금액 = ST조제료.Where(코드==Z코드 AND 적용일ID==해당적용일 AND 시작일<=투약일수 AND 종료일>=투약일수)
         → P점수 또는 D점수 선택
         → (기준금액 × 점수).사사오입
```
