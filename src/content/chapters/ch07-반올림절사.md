# CH07. 반올림/절사 규칙 통합 명세

> **문서 목적:** 약제비 계산 파이프라인에서 사용되는 모든 반올림/절사/올림 규칙을 하나의 문서로 통합한다. 이 문서만으로 `RoundingHelper` 유틸리티 클래스를 구현할 수 있어야 한다.
>
> **출처:** 공단 규칙(15개), 비즈팜 12단계 절사체인, 유팜 MathHelper 9종, EDB NPayRoundType 6종, docs/약국_약제비_계산_분석.md, docs/약국_약제비_계산_로직.md

---

## 1. 개요 --- 반올림이 중요한 이유

약제비 청구에서 **1원의 차이**가 심사 오류(삭감)로 이어진다. HIRA(건강보험심사평가원)는 청구 명세서의 모든 금액 필드를 기계적으로 검증하며, 반올림 방식이 단 한 곳이라도 틀리면 청구 전체가 반려될 수 있다.

**핵심 문제:**
- 계산 단계마다 적용해야 하는 반올림 방식(사사오입/절사/올림)과 단위(1원/10원/100원)가 **모두 다르다**
- 보험유형(건강보험/의료급여/보훈/자보)에 따라 **같은 필드**에도 다른 절사 규칙이 적용된다
- 중간 계산 결과를 언제 반올림하고 언제 유지하느냐가 최종값에 영향을 준다
- `x * 0.1`과 `x / 10`은 부동소수점 오차로 인해 다른 결과를 낼 수 있다

**법적 근거:** 국민건강보험법 제107조, 국고금관리법 제47조 (끝수계산 규정)

## 2. 통합 반올림 규칙 매트릭스

### 2-1. 계산 단계별 반올림 규칙

아래 표는 약제비 계산 파이프라인의 각 단계에서 어떤 반올림을 적용하는지 정리한 것이다.

| # | 계산 단계 | 반올림 방식 | 단위 | 의사코드 | 비고 |
|---|-----------|------------|------|----------|------|
| R01 | **약품 단가 결정** | 사사오입 (4사5입) | 원미만 | `round(단가)` | 실구입가 ≤ 상한금액. 최소 1원 |
| R02 | **1회투약량 라운딩** | 사사오입 (4사5입) | 소수5자리→4자리 | `round(qty, 4, AwayFromZero)` | 소수점 5자리에서 반올림 |
| R03 | **1일투약횟수 라운딩** | 사사오입 (4사5입) | 소수3자리→2자리 | `round(freq, 2, AwayFromZero)` | 소수점 3자리에서 반올림 |
| R04 | **약품별 금액** | 사사오입 (4사5입) | 원미만 | `round(단가 × 1회투약량 × 1일투약횟수 × 총투약일수)` | 각 약품 줄마다 개별 적용 |
| R05 | **조제료 점수 중간값** | 사사오입 (4사5입) | 소수2자리 | `Math.Round(값, 2, AwayFromZero)` | 가산 적용 후 점수 |
| R06 | **조제료 항목 금액** | 사사오입 (4사5입) | 원미만 | `round(기준금액 × 최종점수)` | `.To원단위사사오입()` |
| R07 | **토요가산 분리계산** | 사사오입 (4사5입) | 원미만 | `round(기준금액 × 토요가산점수)` | 기본금액과 별도 계산 후 합산 |
| R08 | **할증 적용** | 사사오입 (4사5입) | 원미만 | `round(금액 × (1 + 할증률/100))` | 자보/비급여 |
| R09 | **사용장려금 단가** | 사사오입 (4사5입) | 원미만 | `round(상한금액 × 10%)`, 최소 1원 | 퇴장방지의약품 직접조제만 |
| R10 | **사용장려금 금액** | 사사오입 (4사5입) | 원미만 | `round(단가 × 1회투약량 × 횟수 × 일수)` | R04와 동일 방식 |
| R11 | **대체조제 장려금 단가** | 사사오입 (4사5입) | 원미만 | `round(약가차액 × 30%)` | 소수점 이하 반올림 |
| R12 | **요양급여비용총액1** | 절사 (내림) | 10원미만 | `trunc10(01항 + 02항)` | A/B/D/E/U/V/W항 제외 |
| R13 | **요양급여비용총액2** | 절사 (내림) | 10원미만 | `trunc10(총액1 + 100/100본인부담금총액)` | |
| R14 | **100/100본인부담금총액** | 절사 (내림) | 10원미만 | `trunc10(U항_합계)` | |
| R15 | **100/100미만 총액** | 절사 (내림) | 10원미만 | `trunc10(A + B + D + E항 합계)` | 선별급여 |
| R16 | **100/100미만 본인일부부담금** | 절사 (내림) | 10원미만 | `trunc10(A×50% + B×80% + D×30% + E×90%)` | 합산 후 1회 절사 (개별X) |
| R17 | **100/100미만 청구액** | 절사 (내림) | 10원미만 | `trunc10(R15 - R16)` | |
| R18 | **청구액** | 절사 없음 | — | `총액1 - 본인일부부담금` | 그대로 사용 |

### 2-2. 보험유형별 본인부담금 절사 규칙

본인부담금의 절사 단위는 보험유형에 따라 다르다. 이것이 가장 자주 실수하는 부분이다.

| 보험유형 코드 | 보험유형 | 본인부담금 절사 | 의사코드 | 근거 |
|--------------|----------|----------------|----------|------|
| **C** (건강보험) | 건강보험 일반 (30%) | 100원 미만 절사 | `trunc100(총액1 × 30%)` | EDB/유팜 코드 확인 |
| **C** (건강보험) | 건강보험 직접조제 (40%) | 100원 미만 절사 | `trunc100(총액1 × 40%)` | 동일 |
| **C** (건강보험) | 건강보험 본인부담률 100% | **10원 미만 절사** | `trunc10(총액1 × 100%)` | EDB: Truncate(x/10)*10 확인 |
| **D** (의료급여) | 의료급여 1종 | 10원 미만 절사 또는 정액 | `trunc10(…)` 또는 `500원/1,000원` | EDB/유팜 코드 확인 |
| **D** (의료급여) | 의료급여 2종 | 10원 미만 절사 또는 정액 | `trunc10(…)` 또는 `1,000원` | 동일 |
| **G** (보훈) | 보훈 100/100 (국비) | — | 본인부담 없음 (전액 국비) | 전소스 일치 |
| **G** (보훈) | 보훈 감면 (30~90%) | **10원 미만 절사** | `trunc10(잔여액 × 부담률)` | 유팜: To십원미만내림() 확인 |
| **G** (보훈) | 보훈 일반 (<100%) | 100원 미만 절사 | `trunc100(…)` | EDB: Truncate(x/100)*100 확인 |
| **F** (자보/산재) | 자동차보험/산재 | 없음 (전액 보험사) | 본인부담 0원 | 자동차손해배상보장법 |
| **E** (공무상) | 공무상 요양급여 | 0원 | 전액 기관 부담 | — |

> **핵심 차이:**
> - 건강보험(C) 일반(30%) → **100원 절사**, 본인부담률 100% → **10원 절사**
> - 의료급여(D) → **10원 절사**
> - 보훈 감면(30~90%) → **10원 절사**, 보훈 일반 → **100원 절사**
> - 근거: 유팜 `To십원미만내림()`/`To백원미만내림()` + EDB `Truncate(x/10)*10`/`Truncate(x/100)*100` 코드 직접 확인 (2025.04.03 재검증)

### 2-3. 전체 파이프라인 흐름도 (반올림 관점)

```
[약품 단가] ──round1(원미만 사사오입)──→ 정수 단가
     ↓
[약품별 금액] ──round1(원미만 사사오입)──→ 정수 약가
     ↓
[조제료 점수] ──round2(소수2자리 사사오입)──→ 소수2자리 점수
     ↓
[조제료 금액] ──round1(원미만 사사오입)──→ 정수 조제료
     ↓
[01항 합산 + 02항 합산]
     ↓
[요양급여비용총액1] ──trunc10(10원미만 절사)──→ 10원 단위 총액
     ↓
[본인부담금 산정]
  ├── 건보(C): ──trunc100(100원미만 절사)──→ 100원 단위
  ├── 의료급여(D): ──trunc10(10원미만 절사)──→ 10원 단위
  └── 보훈감면(G): ──trunc100(100원미만 절사)──→ 100원 단위
     ↓
[청구액] = 총액1 - 본인부담금 (절사 없음)
```

## 3. 반올림 유틸리티 함수 명세

아래 5개 + 2개(보조) = **7개 핵심 함수**로 모든 반올림 규칙을 커버할 수 있다.

### 3-1. `Round1` --- 사사오입 (원미만)

**용도:** 약품 금액, 조제료 금액, 할증 금액 등 대부분의 금액 계산

```
함수명: Round1(x: decimal) → long
설명:   원 미만에서 사사오입 (0.5 이상 올림, 미만 버림)
공식:   Math.Round(x, 0, MidpointRounding.AwayFromZero)
        또는 (long)(x + 0.5m)  ← 양수만 다루는 경우

입력 예시:
  Round1(2302.488)  → 2302
  Round1(435.68)    → 436
  Round1(188.56)    → 189
  Round1(0.4)       → 0
  Round1(0.5)       → 1    ← AwayFromZero이므로 올림
  Round1(1.5)       → 2    ← 은행원반올림(ToEven)이 아님!
```

> **주의:** C#의 `Math.Round(x)` 기본값은 `MidpointRounding.ToEven` (은행원반올림)이다. 반드시 `MidpointRounding.AwayFromZero`를 명시해야 한다. EDB 소스의 `(int)(x + 0.5m)` 패턴은 양수에서 동일한 결과를 내지만, 음수 입력에 대해서는 동작이 다르다. 약제비 계산은 항상 양수이므로 실질적 차이는 없으나, 명시적으로 `AwayFromZero`를 사용하는 것을 권장한다.

### 3-2. `RoundN` --- 사사오입 (소수 N자리)

**용도:** 조제료 점수 중간값(2자리), 1회투약량(4자리), 1일투약횟수(2자리)

```
함수명: RoundN(x: decimal, decimals: int) → decimal
설명:   소수점 N자리에서 사사오입
공식:   Math.Round(x, decimals, MidpointRounding.AwayFromZero)

입력 예시:
  RoundN(4.825, 2)     → 4.83
  RoundN(0.03571, 4)   → 0.0357    ← 1회투약량 (5자리→4자리)
  RoundN(2.335, 2)     → 2.34
```

### 3-3. `Trunc10` --- 10원 미만 절사

**용도:** 요양급여비용총액1/2, 100/100 관련 총액, 의료급여 본인부담금

```
함수명: Trunc10(x: decimal) → long
설명:   10원 미만을 버림 (내림)
공식:   (long)(Math.Floor(x / 10)) * 10
        또는 (long)(x / 10) * 10  ← x가 양수인 경우 동일

입력 예시:
  Trunc10(28789)  → 28780
  Trunc10(4564)   → 4560
  Trunc10(9)      → 0
  Trunc10(10)     → 10
  Trunc10(15)     → 10
  Trunc10(28780)  → 28780   ← 이미 10원 단위면 변화 없음
```

> **주의 (부동소수점):** `x * 0.1`과 `x / 10`은 부동소수점 연산에서 다른 결과를 낼 수 있다. 비즈팜 소스에서 `x * 0.1` 패턴이 발견되었는데, 이는 잠재적 버그 원인이다. **반드시 정수 나눗셈 또는 decimal 타입을 사용할 것.**

### 3-4. `Trunc100` --- 100원 미만 절사

**용도:** 건강보험 본인부담금, 보훈 본인부담금

```
함수명: Trunc100(x: decimal) → long
설명:   100원 미만을 버림 (내림)
공식:   (long)(Math.Floor(x / 100)) * 100
        또는 (long)(x / 100) * 100  ← x가 양수인 경우 동일

입력 예시:
  Trunc100(8634)   → 8600
  Trunc100(9231)   → 9200
  Trunc100(99)     → 0
  Trunc100(100)    → 100
  Trunc100(1050)   → 1000
```

### 3-5. `Ceil10` --- 10원 미만 올림

**용도:** 보훈청구 금액 (유팜 확인), 일부 비급여 절사 옵션

```
함수명: Ceil10(x: decimal) → long
설명:   10원 미만을 올림
공식:   (long)(Math.Ceiling(x / 10)) * 10

입력 예시:
  Ceil10(4561)  → 4570
  Ceil10(4560)  → 4560   ← 이미 10원 단위면 변화 없음
  Ceil10(1)     → 10
```

> **유팜 제출물 확인:** 유팜은 보훈청구 금액에 10원 올림(`Ceil10`)을 적용한다. 공단 규칙에는 이 항목이 명시되지 않았으므로, **약국 현장 확인** 필요 항목이다.

### 3-6. `Round10` --- 10원 미만 사사오입

**용도:** 조제료 단가 (점수 × 점수당단가), 비급여 절사옵션 중 "원단위 절사"

```
함수명: Round10(x: decimal) → long
설명:   10원 미만에서 사사오입
공식:   (long)(Math.Round(x / 10, 0, MidpointRounding.AwayFromZero)) * 10

입력 예시:
  Round10(785.975) → 790    ← 조제료 단가 예시 (의료급여)
  Round10(1745)    → 1750
  Round10(1744)    → 1740
```

> **유팜 명칭 버그와의 관계:** 유팜 MathHelper의 `To원단위사사오입()`이 실제로는 10원 단위에서 사사오입하고 있었다. 이 함수(`Round10`)가 그 실제 동작에 해당한다. §6 소스별 비교표 참조.

### 3-7. `Round100` --- 100원 미만 사사오입

**용도:** 비급여 절사옵션 중 "십원단위 반올림"

```
함수명: Round100(x: decimal) → long
설명:   100원 미만에서 사사오입
공식:   (long)(Math.Round(x / 100, 0, MidpointRounding.AwayFromZero)) * 100

입력 예시:
  Round100(1740) → 1700
  Round100(1750) → 1800
  Round100(1760) → 1800
```

> **유팜 명칭 버그와의 관계:** 유팜 MathHelper의 `To백원단위사사오입()`이 실제로는 1000원 단위에서 사사오입하고 있었다. 새 시스템에서는 함수명과 실제 동작을 반드시 일치시킬 것. §6 참조.

### 3-8. 보조 함수: `Ceil100` --- 100원 미만 올림

**용도:** 비급여 절사옵션 중 "십원단위 올림"

```
함수명: Ceil100(x: decimal) → long
설명:   100원 미만을 올림
공식:   (long)(Math.Ceiling(x / 100)) * 100

입력 예시:
  Ceil100(1740) → 1800
  Ceil100(1700) → 1700
```

### 3-9. 함수 요약표

| 함수명 | 방식 | 단위 | C# 핵심 코드 | 주 용도 |
|--------|------|------|--------------|---------|
| `Round1` | 사사오입 | 원미만 | `Math.Round(x, 0, AwayFromZero)` | 약품금액, 조제료금액 |
| `RoundN` | 사사오입 | 소수N자리 | `Math.Round(x, n, AwayFromZero)` | 점수중간값, 투약량 |
| `Round10` | 사사오입 | 10원미만 | `Math.Round(x/10, 0, AwayFromZero)*10` | 조제료단가(의료급여) |
| `Round100` | 사사오입 | 100원미만 | `Math.Round(x/100, 0, AwayFromZero)*100` | 비급여옵션 |
| `Trunc10` | 절사(내림) | 10원미만 | `Math.Floor(x/10)*10` | 총액, 의료급여본인부담 |
| `Trunc100` | 절사(내림) | 100원미만 | `Math.Floor(x/100)*100` | 건보본인부담 |
| `Ceil10` | 올림 | 10원미만 | `Math.Ceiling(x/10)*10` | 보훈청구(유팜) |
| `Ceil100` | 올림 | 100원미만 | `Math.Ceiling(x/100)*100` | 비급여옵션 |

## 4. 비급여 반올림 옵션 (NPayRoundType 6종)

비급여 금액은 약국 설정에 따라 절사 방식을 선택할 수 있다. 급여 계산에는 영향 없다.

### 4-1. EDB 기준 NPayRoundType

EDB 소스에서 정의된 6종의 비급여 반올림 타입:

| 타입값 | 명칭 | 동작 | 예시 (1,745원) | 함수 매핑 |
|--------|------|------|---------------|-----------|
| 0 | 원단위 절사 | 10원 미만 버림 | 1,740 | `Trunc10` |
| 1 | 십원단위 버림 | 100원 미만 버림 | 1,700 | `Trunc100` |
| 2 | 십원단위 반올림 | 100원 미만 사사오입 | 1,700 | `Round100` |
| 3 | 십원단위 올림 | 100원 미만 올림 | 1,800 | `Ceil100` |
| 4 | 절사없음 (원단위) | 그대로 사용 | 1,745 | 없음 (identity) |
| 5 | 원미만 사사오입 | 원 미만 반올림 | 1,745 | `Round1` |

> **주의 (명칭 혼동):** "원단위 절사"(타입0)는 "원 단위에서 절사"가 아니라 "10원 미만을 절사하여 **원(10원)단위로 맞춤**"이라는 뜻이다. 명칭만 보면 오해하기 쉬우므로, 구현 시 타입값 → 함수 매핑을 명확히 할 것.

### 4-2. EDB 본인부담 절사 분기 로직

EDB 소스에서 보험유형 코드별로 적용하는 본인부담 절사 규칙:

```
switch (보험유형) {
    case 'C':  // 건강보험
        본인부담금 = Trunc100(…)     // 100원 미만 절사
        break;
    case 'D':  // 의료급여
        본인부담금 = Trunc10(…)      // 10원 미만 절사
        break;
    case 'G':  // 보훈
        if (본인부담률 == 100%) {
            본인부담금 = Trunc10(…)   // 100/100 → 10원 미만 절사
        } else {
            본인부담금 = Trunc100(…)  // 100/100 미만 → 100원 미만 절사
        }
        break;
}
```

### 4-3. EDB 날짜 분기 규칙

EDB 소스에서 발견된 날짜별 반올림 규칙 변경:

| 분기점 | 코드 | 변경 내용 |
|--------|------|-----------|
| C31/C32 날짜 분기 | 약가 계산 | 특정 날짜 이후 약가 반올림 방식 변경 (상세 확인 필요) |
| M20 날짜 분기 | 수가 계산 | 특정 날짜 이후 수가 절사 방식 변경 (상세 확인 필요) |
| MpvaPrice 역산 | 역산 로직 | 단가 역산 시 반올림 적용 순서 변경 |

> 이 날짜 분기들은 과거 정책 변경 이력이며, 현행 시스템에서는 최신 규칙만 적용하면 된다. 그러나 과거 명세서 재검증 기능을 구현할 경우 날짜 분기가 필요하다.

## 5. 수납 절사 옵션 (gSelfAmt 3종)

비즈팜 소스에서 발견된 약국별 수납 시 최종 절사 옵션. 환자가 실제 수납하는 금액에 적용된다.

| gSelfAmt 값 | 의미 | 동작 | 예시 |
|-------------|------|------|------|
| 0 | 절사 없음 | 계산된 금액 그대로 수납 | 8,634 → 8,634 |
| 1 | 10원 미만 절사 | 1의 자리 버림 | 8,634 → 8,630 |
| 2 | 10원 미만 사사오입 | 1의 자리에서 반올림 | 8,634 → 8,630 / 8,636 → 8,640 |

> **주의:** gSelfAmt는 **약국 수납 단계**의 절사 옵션이며, HIRA 청구 금액에는 영향을 주지 않는다. 청구 금액은 §2의 규칙대로 계산하고, 수납 금액만 약국 설정에 따라 조정한다. 차액은 약국이 부담/이익 처리한다.

## 6. 소스별 비교표 + 불일치 판정

4개 소스(공단/유팜/비즈팜/EDB)에서 동일 규칙을 어떻게 표현하는지 비교하고, 불일치 여부를 판정한다.

### 6-1. 약품 금액 반올림

| 항목 | 공단 | 유팜 | 비즈팜 | EDB | 판정 |
|------|------|------|--------|-----|------|
| 약가(단가×수량) | 원미만 4사5입 | `Round1` | `round()` | `(int)(x + 0.5m)` | **동일** --- 모두 원미만 사사오입 |

> EDB의 `(int)(x + 0.5m)`은 양수에서 `Math.Round(x, 0, AwayFromZero)`와 동일 결과. 약제비는 항상 양수이므로 실질적으로 같다.

### 6-2. 조제료(수가) 반올림

| 항목 | 공단 | 유팜 | 비즈팜 | EDB | 판정 |
|------|------|------|--------|-----|------|
| 수가(점수×단가) | 10원미만 4사5입 | "사사오입" | 사사오입 | `(int)` 절사 | **불일치** |

> **중요 차이:**
> - 공단 규칙: 조제료 단가(점수×점수당단가)는 "10원 미만 4사5입" (의료급여 기준)
> - 유팜: "사사오입" (원미만 사사오입으로 추정, §6-3 명칭버그 참조)
> - EDB: `(int)` 캐스팅 = **절사** (소수점 이하 버림)
>
> **docs/ 분석 결론:** 조제료 각 항목 금액은 **원미만 사사오입**(`Round1`)이 정규칙이다. EDB의 `(int)` 절사는 버그이거나 다른 컨텍스트일 수 있다. 의료급여 환경에서의 조제료 단가가 10원 단위 사사오입(`Round10`)인 경우도 현장에서 확인된 바 있으므로, 보험유형별 분기가 필요할 수 있다.

### 6-3. 유팜 명칭 버그 상세

| 유팜 메서드명 | 유팜이 주장하는 동작 | 실제 동작 | 공단 규칙과의 매칭 |
|--------------|--------------------|-----------|--------------------|
| `To원단위사사오입()` | 원 단위에서 사사오입 | **10원 단위**에서 사사오입 (`Round10`) | 공단 "10원미만4사5입"과 **동일** |
| `To백원단위사사오입()` | 100원 단위에서 사사오입 | **1000원 단위**에서 사사오입 | 명칭과 실제가 **불일치** |

> **새 시스템 요구사항:** 함수명에 반드시 **실제 절사 대상 단위**를 명시할 것. `Round10`은 "10원 미만을 사사오입"이라는 의미가 명확하다.

### 6-4. 본인부담금 절사

| 항목 | 공단 | 유팜 | 비즈팜 | EDB | 판정 |
|------|------|------|--------|-----|------|
| 건보 본인부담금 | 100원미만 절사 | `Trunc100` | `CalSelfAmt` 4종혼재 | C: 100원 절사 | **동일** (비즈팜은 혼재) |
| 의료급여 본인부담금 | 10원미만 절사 | 10원 절사 | CalSelfAmt | D: 10원 절사 | **동일** |
| 보훈 본인부담금 | 100원미만 절사 | — | `Int(x*율+0.5)*10` | G(100/100→10원, <100%→100원) | **비즈팜 주의** |

> **비즈팜 보훈 공식 `Int(x*율+0.5)*10`의 의미:**
> `Int(x * 율 + 0.5) * 10`은 사실상 10원 단위 사사오입(`Round10`)이다. 이는 공단 규칙의 100원 절사(`Trunc100`)와 **다른 결과**를 낸다.
>
> 예시: x=20,000, 율=0.3 → Int(6000.5)*10 = 60,010 vs Trunc100(6000) = 6,000
>
> 이 차이는 비즈팜 버그 목록(§7)에 포함된다.

### 6-5. 총액 절사

| 항목 | 공단 | 유팜 | 비즈팜 | EDB | 판정 |
|------|------|------|--------|-----|------|
| 요양급여비용총액1 | 10원미만 절사 | `Trunc10` | 12단계체인 내 | 10원 절사 | **동일** |
| 요양급여비용총액2 | 10원미만 절사 | `Trunc10` | 12단계체인 내 | 10원 절사 | **동일** |

### 6-6. 보훈청구 반올림

| 항목 | 공단 | 유팜 | 비즈팜 | EDB | 판정 |
|------|------|------|--------|-----|------|
| 보훈청구액 반올림 | 명시 없음 | 10원 올림 (`Ceil10`) | `Int(x*율+0.5)*10` | — | **불일치 --- 현장 확인 필요** |

> 공단 규칙에는 보훈청구액의 반올림 방식이 명확히 기술되지 않았다. 유팜은 올림, 비즈팜은 10원 사사오입을 적용하고 있어 서로 다르다. 현장 확인이 필요한 항목이다.

## 7. 주의사항 --- 비즈팜 버그 목록 (새 시스템에서 반복 금지)

비즈팜 소스에서 발견된 반올림 관련 버그 5건. 새 시스템에서는 이 패턴을 반복하지 않아야 한다.

### 버그 #1: `x * 0.1` vs `x / 10` 혼재

```
문제: 동일한 "10원 절사" 로직을 구현할 때 x * 0.1과 x / 10을 혼용
원인: 부동소수점 연산에서 0.1은 정확히 표현되지 않음
영향: 특정 금액에서 1원 차이 발생 가능
수정: decimal 타입 사용 시 x / 10m으로 통일, 또는 정수 나눗셈 (x / 10) * 10 사용
```

### 버그 #2: `CalSelfAmt` 4종 혼재

```
문제: 본인부담금 계산 함수가 4가지 다른 절사 방식을 혼재하여 사용
원인: 보험유형별 분기가 함수 내부에 산재되어 있음
영향: 특정 보험유형에서 잘못된 절사 함수 호출
수정: 보험유형별로 명확히 분리된 절사 함수 매핑 테이블 사용 (§2-2 참조)
```

### 버그 #3: 보훈 본인부담 `Int(x*율+0.5)*10`

```
문제: 보훈 본인부담금을 10원 단위 사사오입으로 계산 (Bohun_Amt = Int(x * 율 + 0.5) * 10)
정규칙: 보훈 감면 본인부담금은 100원 미만 절사 (Trunc100)
영향: 보훈 환자 본인부담금이 최대 90원까지 차이남
수정: Trunc100() 함수 사용으로 통일
```

### 버그 #4: 12단계 절사체인 내 순서 의존성

```
문제: 12단계에 걸친 순차적 절사 과정에서 중간 단계의 절사 순서가 결과에 영향
원인: 각 단계의 절사를 누적 적용하면서, 어떤 단계에서는 절사 전 값을 사용하고
      어떤 단계에서는 절사 후 값을 사용하는 등 일관성 없음
영향: 복잡한 처방(다수 약품 + 선별급여 + 보훈)에서 최종값 불일치
수정: §2-1 매트릭스의 순서를 엄격히 따르되, 각 단계의 입력이 이전 단계의
      절사 후 값인지 원본값인지를 명확히 문서화
```

### 버그 #5: 비급여 절사와 급여 절사 분리 미흡

```
문제: gSelfAmt(수납 절사)가 급여 청구 금액 계산에도 영향을 미치는 경로 존재
정규칙: gSelfAmt는 수납 단계에서만 적용, 청구 금액에는 영향 없음
영향: 특정 gSelfAmt 설정에서 청구 금액이 미세하게 변동
수정: 청구 파이프라인과 수납 파이프라인을 완전 분리
```

## 8. 100/100미만(선별급여) 절사 특별 규칙

선별급여 본인부담금 계산 시 주의할 절사 규칙:

```
100/100미만_본인일부부담금 계산:
  A항_부담 = A항_합계 × 50%
  B항_부담 = B항_합계 × 80%
  D항_부담 = D항_합계 × 30%
  E항_부담 = E항_합계 × 90%

  // ★ 핵심: 각 항별로 개별 절사하지 않는다!
  // 전체 합산 후 1회만 절사한다.
  100/100미만_본인일부부담금 = Trunc10(A항_부담 + B항_부담 + D항_부담 + E항_부담)
```

> **잘못된 구현 (흔한 실수):**
> ```
> // ✗ 이렇게 하면 안 됨
> Trunc10(A항_합계 × 50%) + Trunc10(B항_합계 × 80%) + ...
> ```
> 개별 절사 후 합산하면 최대 수십 원의 차이가 발생한다.

**연간 외래진료 365회 초과 특례:**
- 연간 외래진료 횟수가 365회를 초과하는 경우:
- `100/100미만_본인일부부담금 = Trunc10(100/100미만_총액 × 90%)`
- 위의 A/B/D/E 개별 부담률 대신 일괄 90% 적용
