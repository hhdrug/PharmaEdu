## 1. 개요

약품금액(약가)은 약제비의 핵심 구성요소로, 처방전에 기재된 개별 약품에 대해 **단가 × 투약량 × 횟수 × 일수**로 산출한다. 이 챕터는 "개별 약품 1줄의 금액을 얼마로 결정하는가"에 집중하며, 조제료(CH02/CH03)·본인부담금(CH05)·3자배분(CH06)은 별도 챕터에서 다룬다.

**계산 파이프라인 위치:**

```
[입력: 약품 목록] → ★ CH01: 개별 약품금액 산출 ★ → [항별 분류·합산] → [조제료] → [총액] → [본인부담] → [청구액]
```

**이 챕터에서 결정되는 것:**
- 개별 약품의 단가 (어떤 가격을 사용할 것인가)
- 개별 약품의 금액 (단가 × 투약량 × 횟수 × 일수, 반올림 포함)
- 약품의 급여구분별 분류 (어느 항에 배치할 것인가)
- 약품의 내복/외용/주사 분류
- 저가대체장려금·사용장려금 금액

## 2. 약품금액 산출 공식

### 2-1. 통합 공식

4개 소스 모두 동일한 기본 공식을 사용한다:

```
약품금액 = round_half_up(단가 × 1회투약량 × 1일투여횟수 × 총투여일수)
```

| 소스 | 구현 코드 | 검증 결과 |
|------|-----------|----------|
| `[비즈팜]` | `Int(One_Qty * IN_Qty * Tot_Cnt * Danga + 0.5)` | VB6 Int()는 음수방향 절사이므로 양수에서 +0.5 = 4사5입 |
| `[공단]` | "단가 × 1회투약량 × 1일투여횟수 × 총투여일수, 원미만 4사5입" | 고시 원문 |
| `[유팜]` | `(단가 + 저가대체금) × (1 + 할증률/100) × 소모량 → To원미만사사오입()` | 할증·저가대체 포함 확장형 |
| `[EDB]` | `(int)(DOSE * DNUM * DDAY / PACK * PRICE + 0.5m)` | 팩수량 나누기 포함 |
| `[docs]` | "금액 = 단가 × 1회투약량 × 1일투약횟수 × 총투약일수 → 원미만 4사5입" | 정리본 |

**통합 정규 공식:**

```python
def calc_drug_amount(단가: int, 1회투약량: Decimal, 1일투여횟수: Decimal,
                     총투여일수: int, 팩수량: int = 0,
                     저가대체장려금단가: Decimal = 0, 할증률: Decimal = 0) -> int:
    """
    약품 1줄의 금액을 계산한다.

    Args:
        단가: 결정된 단가 (원, 정수). §3에서 결정.
        1회투약량: 소수점 4자리까지 (5자리에서 4사5입)
        1일투여횟수: 소수점 2자리까지 (3자리에서 4사5입)
        총투여일수: 정수
        팩수량: 0이면 일반, >1이면 팩단위 약품
        저가대체장려금단가: §6에서 산출된 장려금 단가 (원)
        할증률: 자보/비보험 할증률 (%, 예: 30이면 30%)

    Returns:
        약품금액 (정수, 원)
    """
    # 소모량 계산
    if 팩수량 > 1:
        소모량 = 1회투약량 * 1일투여횟수 * 총투여일수 / 팩수량
    else:
        소모량 = 1회투약량 * 1일투여횟수 * 총투여일수

    # 할증 적용 (자보/비보험만)
    유효단가 = (단가 + 저가대체장려금단가) * (1 + 할증률 / 100)

    # 최종 금액 = 원미만 4사5입
    금액 = round_half_up(유효단가 * 소모량)  # MidpointRounding.AwayFromZero

    return int(금액)
```

### 2-2. 소수점 정밀도 규칙

| 항목 | 유효 자릿수 | 반올림 규칙 | 소스 |
|------|------------|------------|------|
| **1회투약량** | 소수점 **4자리**까지 | 5자리에서 4사5입 | `[공단]` `[docs]` |
| **1일투여횟수** | 소수점 **2자리**까지 | 3자리에서 4사5입 | `[공단]` `[docs]` |
| **총투여일수** | 정수 | - | 전체 |
| **단가** | 정수 (원) | 원미만 4사5입 | `[공단]` `[docs]` `[비즈팜]` |
| **금액 (최종)** | 정수 (원) | 원미만 4사5입 | 전체 (4소스 일치) |

```python
# 전처리 예시
1회투약량 = round_half_up(raw_1회투약량, 4)   # 0.33333 → 0.3333
1일투여횟수 = round_half_up(raw_1일투여횟수, 2)  # 2.333 → 2.33
```

### 2-3. 팩수량 처리

팩단위 약품(예: 크리멘정 28정/팩)은 1회투약량을 팩 기준으로 환산한다.

**방법 1 — 1회투약량 환산 (`[공단]` `[docs]`):**
```
예: 28정/팩, 1일 1정, 28일분
1회투약량 = 1/28 = 0.0357 (소수점 5자리에서 4사5입)
금액 = 8,626 × 0.0357 × 1 × 28 = 8,626원
```

**방법 2 — 소모량 나누기 (`[EDB]`):**
```
PD_SUM = (int)(DOSE * DNUM * DDAY / PACK * PRICE + 0.5)
예: (1 * 1 * 28 / 28 * 8626 + 0.5) = 8,626원
```

**방법 3 — Get소모량 (`[유팜]`):**
```csharp
소모량 = 용량 × 횟수 × 일수 ÷ 팩수량
금액 = 단가 × 소모량 → To원미만사사오입()
```

> **구현 권고:** 방법 1(1회투약량 환산)을 기본으로 하되, 검증 시 방법 2의 결과와 일치하는지 확인.

### 2-4. 반올림 함수 명세

```python
def round_half_up(value: Decimal, digits: int = 0) -> Decimal:
    """
    4사5입 (MidpointRounding.AwayFromZero).
    0.5를 무조건 올림. Banker's rounding(짝수 반올림)이 아님.

    비즈팜: Int(x + 0.5)  — 양수에서 동일 효과
    EDB: (int)(x + 0.5m)  — 동일
    유팜: Math.Round(x, MidpointRounding.AwayFromZero)
    공단: "원미만 4사5입"

    ★ 단가가 1원 미만이면 1원으로 올린다.
    """
    result = Decimal(value).quantize(Decimal(10) ** -digits, rounding=ROUND_HALF_UP)
    if digits == 0 and result < 1:
        result = 1
    return result
```

### 2-5. 특수 투약량 케이스

| 케이스 | 1회투약량 | 1일투여횟수 | 총투여일수 | 비고 |
|--------|----------|------------|-----------|------|
| (a) 일반 경구약 | 정수 (예: 1) | 정수 (예: 3) | 정수 (예: 7) | 기본 |
| (b) 1/3정 → 1일 1정 | 0.3333 또는 1 | 3 또는 1 | 7 | 명세서에는 1일투약량 '1'로 기재 가능 |
| (c) 동일약 기간별 용법 상이 | 각각 별도 줄 | 각각 별도 줄 | 각각 별도 | 예: 1정-2회-7일 / 2정-1회-21일 |
| (d) 동일약 동일날 용법 상이 | 각각 별도 줄 | 각각 별도 줄 | 동일 | 예: 아침 1정 / 저녁 0.5정 |
| (e) 외용약 (점안액/연고) | 1 (병) | 1 | 1 | 병/개 단위. 실투약일수는 요양급여일수에 별도 산입 |
| (f) 파스류 격일 | 총투약량 (5) | 1 | 1 | 격일×10일 → 5매를 1회투약량으로 |
| (g) 팩단위 | 1/팩수량 (0.0357) | 1 | 28 | 상기 §2-3 참조 |
| (h) 시럽제 소분 | 5 (mL) | 3 | 3 | 최소단위(1mL)당 상한금액 × 5 × 3 × 3 |

## 3. 단가 결정 로직

### 3-1. 단가 우선순위 (3중 분기)

4개 소스를 통합하면, 약품 단가는 다음 우선순위로 결정된다:

```
[1단계] 약제 급여 목록(PPREVALU/보험등재 DB) 조회
        → 처방일자 기준 유효기간 내 보험단가(insurance_amt/상한금액) 존재?

[2단계] 존재하면 → 보험등재 상태(Del_Yn) 및 보험유형별 분기
        존재하지 않으면 → 판매단가(ilban_amt) 폴백

[3단계] 실거래가 비교 (유팜 전용, WAPPid 있을 때)
```

**통합 단가 결정 의사코드:**

```python
def determine_unit_price(drug, patient, 처방일자) -> (int, str):
    """
    단가와 급여구분을 결정한다.

    Returns:
        (단가, 급여구분)  # 급여구분: "보험"/"비급여"/"전액본인부담" 등
    """
    # ──── 1단계: 보험등재 DB 조회 ────────────────────────────
    보험약가_record = query_PPREVALU(drug.약품코드, 처방일자)
    #   SELECT insurance_amt, Del_Yn, Gudan_id
    #   FROM PPREVALU
    #   WHERE drug_id = ? AND Insurance_From <= 처방일자 AND Insurance_to >= 처방일자

    if 보험약가_record is None:
        # 보험 미등재 → 판매단가(일반약가) 사용
        return (drug.판매단가 or drug.ilban_amt, "비급여")

    보험단가 = 보험약가_record.insurance_amt
    Del_Yn = 보험약가_record.Del_Yn

    if 보험단가 == 0:
        # 보험단가 0원 → 판매단가 폴백
        return (drug.판매단가 or drug.ilban_amt, "비급여")

    # ──── 2단계: Del_Yn(보험등재 상태) 분기 ──────────────────
    #  [비즈팜] 7종 분기, [docs] §3-2 참조

    if drug.급여구분 == "비급여":  # UI에서 "비" 선택
        # 비급여 지정 → 판매단가 우선, 없으면 보험단가
        단가 = drug.판매단가 if drug.판매단가 > 0 else 보험단가
        return (단가, "비급여")

    # 보험 약품 분기
    match Del_Yn:
        case None | "" | "M" | "G" | "P":
            # 정상 / 미삭제(경고) / 급여정지(경고) / 100원당100(특수)
            단가 = 보험단가
            if "M": log_warning("미삭제 약품입니다")
            if "G": log_warning("급여정지 약품입니다")
            if "P": set_전액본인부담_flag()

        case "C":  # 산재 전용
            if patient.보험유형 == "산재":
                단가 = 보험단가
            else:
                단가 = drug.판매단가 if drug.판매단가 > 0 else 보험단가

        case "B":  # 의료급여 전용
            if patient.보험유형 == "의료급여":
                단가 = 보험단가
            else:
                단가 = drug.판매단가 if drug.판매단가 > 0 else 보험단가

        case "A":  # 산재 + 의료급여 전용
            if patient.보험유형 in ("산재", "의료급여"):
                단가 = 보험단가
            else:
                단가 = drug.판매단가 if drug.판매단가 > 0 else 보험단가

        case "F":  # 폐기약품
            단가 = drug.판매단가 if drug.판매단가 > 0 else 보험단가
            급여구분 = "비급여"  # 강제 비급여 전환
            return (단가, 급여구분)

    # ──── 3단계: 실거래가 비교 (선택적) ──────────────────────
    # [유팜] WAPPid(외래약국처방전ID) 존재 시
    if drug.WAPPid and drug.WAPPid > 0:
        if drug.보험 or drug.전액본인부담 or drug.보훈보험:
            if 0 < drug.실거래가 < 보험단가:
                단가 = drug.실거래가  # 실거래가가 보험단가보다 낮으면 실거래가
            else:
                단가 = 보험단가

    # ──── 4단계: 약품평균가(DrugAvgValue) 우선적용 (선택적) ───
    # [비즈팜] 환경설정 chkDrugAvgValue="1"일 때
    if 환경설정.약품평균가_사용:
        평균가 = query_PDRUVALU(drug.약품코드)  # Amt4 > Amt3 > Amt2 > Amt1 순
        if 0 < 평균가 < 보험단가:
            단가 = 평균가

    # ──── 최종 보정 ────────────────────────────────────────
    단가 = round_half_up(단가)   # 원미만 4사5입
    if 단가 < 1:
        단가 = 1                 # 최소 1원 [공단] [docs]

    return (단가, "보험")
```

### 3-4. 비보험 약품 단가

비급여로 분류된 약품의 단가 결정:

```python
# [비즈팜] 비급여 약품 단가
if drug.급여구분 == "비급여":
    if drug.판매단가 > 0:
        단가 = drug.판매단가   # ilban_amt 우선
    else:
        단가 = drug.보험단가   # insurance_amt 폴백
```

## 6. 저가대체장려금 / 사용장려금

### 6-1. 저가대체조제장려금 (처방조제)

생물학적 동등성이 확보된 저가 약품으로 대체 조제 시 발생하는 장려금.

```python
def calc_저가대체장려금(처방약_상한금액: int, 대체약_실구입가: int,
                    1회투약량: Decimal, 1일투여횟수: Decimal,
                    총투여일수: int, 내방일: date) -> (int, int):
    """
    Returns:
        (장려금단가, 장려금금액)
    """
    # 적용 조건: 처방조제 AND 보험 AND 2020.03.01 이후
    if 내방일 < date(2020, 3, 1):
        return (0, 0)

    약가차액 = 처방약_상한금액 - 대체약_실구입가
    if 약가차액 <= 0:
        return (0, 0)

    # 장려금 단가 = 차액 × 30% → 원미만 반올림  [유팜] [공단] [docs]
    장려금단가 = round_half_up(약가차액 * Decimal("0.3"))

    # 장려금 금액
    소모량 = 1회투약량 * 1일투여횟수 * 총투여일수
    장려금금액 = round_half_up(장려금단가 * 소모량)

    return (int(장려금단가), int(장려금금액))
```

**명세서 기재:**
- 줄1: 대체약 (조제구분='4', 단가=실구입가)
- 줄2: 처방약 (조제구분='9', 단가=장려금단가)
- 두 줄 모두 **01항**의 해당 목(내복/외용/주사)에 기재

**본인부담금 산정 시:**
```python
# 장려금은 총액에 포함되지만, 본인부담 산정 시 제외
본인부담_기준액 = 요양급여비용총액1 - 장려금금액  # [공단] [docs] [유팜]
본인일부부담금 = trunc100(본인부담_기준액 × 부담률)
# 장려금은 전액 공단부담(청구액)에 포함
```

**계산 예시:**
```
처방약 A정 (상한 1,000원), 대체약 B정 (실구입가 700원)
1회 1정, 1일 2회, 3일분

B정 금액: 700 × 1 × 2 × 3 = 4,200원 (조제구분 4)
장려금 단가: round((1000-700) × 30%) = 90원
장려금 금액: 90 × 1 × 2 × 3 = 540원 (조제구분 9)
```

### 6-2. 퇴장방지의약품 사용장려금 (직접조제)

```python
def calc_사용장려금(상한금액: int, 1회투약량: Decimal,
                 1일투여횟수: Decimal, 총투여일수: int,
                 처방유형: str) -> int:
    """
    처방조제: 산정 불가 (0원)
    직접조제 (의약분업 예외지역): 상한금액 × 10%
    """
    if 처방유형 == "처방조제":
        return 0

    # 직접조제
    사용장려금단가 = round_half_up(상한금액 * Decimal("0.1"))  # 원미만 4사5입
    if 사용장려금단가 < 1:
        사용장려금단가 = 1

    소모량 = 1회투약량 * 1일투여횟수 * 총투여일수
    사용장려금금액 = round_half_up(사용장려금단가 * 소모량)

    return int(사용장려금금액)
```

**명세서 기재:**
- 구입약가 줄 + 사용장려금 줄 (별도)
- 사용장려금 줄의 특정내역 JS002 = "99"
- 사용장려금은 01항의 해당 목(내복/외용/주사)에 기재

**선별급여 퇴장방지의약품인 경우:**
- 구입약가: 해당 항(A/B/D/E/U항)의 각 목에 기재
- 사용장려금: **01항**(약가)의 각 목에 기재 (항번호가 다름에 주의)

### 6-3. 공통 원칙

```

★ 본인부담금 산정 시 장려금은 기준액에서 제외한다
★ 장려금은 전액 청구액(공단부담)에 포함된다
★ 본인부담 제외 원칙은 대체조제장려금·사용장려금 모두 동일
```

## 9. 주의사항

### 9-1. 구현 시 필수 체크 사항

1. **반올림 방식 통일:** 4사5입(AwayFromZero)을 사용할 것. 프로그래밍 언어 기본 반올림이 Banker's rounding(짝수 반올림)인 경우 명시적으로 AwayFromZero를 지정해야 한다.

2. **단가 최소값:** 단가가 0원 또는 1원 미만이면 **1원**으로 올린다. `[공단]` `[docs]` 모두 명시.

3. **금액 0원 방지:** 모든 구성요소(단가, 1회투약량, 1일투여횟수, 총투여일수)가 0보다 커야 유효한 약품이다. 하나라도 0이면 금액 = 0원.

4. **소수점 연산 정밀도:** VB6의 Double/Currency와 C#의 decimal은 정밀도가 다르다. 신규 구현 시 **decimal** 타입을 사용하여 부동소수점 오차를 방지할 것.

5. **팩수량 0 나누기 방지:** 팩수량이 0이면 1로 치환하여 0 나누기 오류를 방지. `[EDB]`에서 확인된 패턴.

6. **Del_Yn 분기 순서:** F(폐기) → A/B/C(보험유형별) → P(100/100) → M/G(경고) → 기타(정상) 순으로 평가. F는 최우선 비급여 전환.
