## 1. 개요

약국 조제료에는 시간대, 요일, 연령, 약품형태 등에 따른 **가산**이 존재한다. 가산은 기본 조제료 점수에 추가 요율을 곱하거나 별도 점수를 더하는 방식으로 산정된다.

가산의 종류는 8가지이며, 이들 사이에는 **상호 배타적 우선순위 규칙**이 존재한다. 잘못된 우선순위로 가산을 적용하면 청구액이 달라지므로, 정확한 if-else 체인 구현이 핵심이다.

## 3. 가산 우선순위 -- 배타적 if-else 체인

### 3-1. 우선순위 규칙 (확정)

```
[전제] 비대면 조제(DrugSafe_YN = "U") → 모든 가산 비활성화, ZC001~004로 전환
                                        ↓ (비대면이 아닌 경우)
1순위: 가루약 가산      → 적용 시 야간/공휴/소아심야/토요/소아 전부 배제
                        ↓ (가루약 아닌 경우)
2순위: 야간/공휴 가산   → 배타적(야간 또는 공휴 중 하나)
3순위: 소아심야 가산    → 2순위 미해당 시
4순위: 토요 가산        → 2~3순위 미해당 시

[추가] 소아 가산: 1순위(가루약) 미적용 시에 한해 2~4순위와 중복 가능
```

### 3-2. 유팜(Upharm) 소스 확인 -- `Base조제료Logic.Get조제료Item()`

```csharp
// 파일: Base조제료Logic.cs, 107~160줄
protected virtual decimal Get조제료Item(
    조제료CodeTypes type, int 최대내복조제일수,
    bool 야간공휴가산여부 = false, bool 소아가산여부 = false,
    bool 소아심야가산여부 = false, bool 토요가산여부 = false,
    bool 가루약가산여부 = false)
{
    if (가루약가산여부)
    {
        num2 = Math.Round(num2 * 보험기준가.가루약가산률, 2, MidpointRounding.AwayFromZero);
        // → 다른 모든 가산 진입 불가 (else 블록 안에 있으므로)
    }
    else
    {
        if (야간공휴가산여부)
        {
            num2 = Math.Round(num2 * (1m + Get공휴가산률(보험기준가)), 2, MidpointRounding.AwayFromZero);
        }
        else if (소아심야가산여부)
        {
            // 2023.11.01 전후 분기
        }
        else if (토요가산여부)
        {
            // 토요 가산 분리 계산
        }
        // ★ 소아 가산은 위 if-else와 독립적으로 적용
        if (소아가산여부)
        {
            num2 += Get기준점수소아가산(보험기준가);
        }
    }
}
```

**핵심 구조:** `가루약` 블록과 `else` 블록이 완전 분리되어, 가루약 적용 시 소아 가산도 진입 불가. 소아 가산은 `else` 블록 내에서 if-else 체인과 독립적(`if`문 별도)으로 위치하여 2~4순위 가산과 중복 가능.

### 3-3. 소아 + 가루약 동시 해당 시 처리

```csharp
// Base조제료Logic.cs, 154~160줄
private bool Check소아가산_조제료(bool 가루약가산여부, bool 소아여부)
{
    if (가루약가산여부 && 소아여부) return false;  // ★ 가루약 + 소아 = 소아 미적용
    return 소아여부;
}
```

## 4. 각 가산별 상세 로직

### 4-1. 야간 가산

#### 판정 조건

| 소스 | 판정 로직 |
|------|-----------|
| **비즈팜** | `Check_Holiday()` 서브루틴 (라인 8535~8640). ZTime1/ZTime2/ZTime3로 기준시간 로딩 후 비교 |
| **EDB** | `Step5_TimeConditionAndChadung()` — `m_isNight = true` 시 `text2 = "1"` |
| **유팜** | `야간공휴가산여부` 파라미터 (ST조제료적용일 테이블 시간 기준) |

**시간 기준 (비즈팜 PZCVALUE 기본값):**

| 코드 | 의미 | 기본값 |
|------|------|--------|
| ZTime1 | 야간가산 시작시간 (영업 시작) | 0900 |
| ZTime2 | 야간가산 종료시간 (토요일) | 1300 |
| ZTime3 | 야간가산 종료시간 (평일) | 1800 |

**야간 판정 분기:**

```
토요일: presTime < 0900 OR presTime >= 1300 → 야간
평일:   presTime < 0900 OR presTime >= 1800 → 야간
공휴일: → 공휴 가산이 우선 (야간 판정 생략)
```

#### 산정코드 접미사

| 대상 코드 | 접미사 | 예시 |
|-----------|--------|------|
| Z2000 (조제기본료) | `010` | Z2000010 |
| Z3000 (복약지도료) | `010` | Z3000010 |
| Z41xx (내복조제료) | `010` | Z4103010 (3일분 야간) |
| Z4120 (외용조제료) | `010` | Z4120010 |

#### 금액 공식

```
최종점수 = 기본점수 × (1 + 공휴가산률)
금액 = (기준금액 × 최종점수).To원단위사사오입()
```

- 처방조제: P공휴가산률 사용
- 직접조제: D공휴가산률 사용
- 라운딩: `Math.Round(값, 2, MidpointRounding.AwayFromZero)`

> **참고:** 야간과 공휴일은 **동일한 가산률**(공휴가산률)을 사용한다. 코드 분기만 다를 뿐 계산식은 동일하다.

#### 비즈팜 Holiday_gb 결합 체계

```
Holiday_gb 값 정의:
  "0"/"N" : 해당 없음
  "1"     : 야간
  "5"     : 공휴일
  "6"     : 6세 미만
  "7"     : 6세 미만 + 공휴일
  "8"     : 6세 미만 + 야간
```

```
IF 나이 < 6세 OR 주민번호앞6자리 = "000000":
    IF Holiday_gb = "1" → "8" (6세 + 야간)
    ELIF Holiday_gb = "5" → "7" (6세 + 공휴일)
    ELSE → "6" (6세 단독)
```

### 4-3. 소아 가산 (6세 미만)

#### 판정 조건

```
조건: 나이 < 6세 OR 주민번호 앞6자리 = "000000"
추가 조건 (유팜): 가루약가산여부 = FALSE (가루약과 소아 동시 시 소아 미적용)
추가 조건 (EDB): 비대면("U")이 아닐 것, 가루약+6세 미만 시 text="0" → 6세 비활성화
```

**EDB 소스 (PrsBillCalcM.SugaCalc.cs, 라인 1436~1445):**

```csharp
// text 결정 (연령 접미사)

{
    text = "6";         // 6세 미만 플래그
    M_Is6Age = true;
}
// ★ 가루약 + 6세미만 → 소아 비활성화
if (M_Ps_Powder_YN != "N" && text == "6" && M_Pay_ATB_Day + M_Npay_ATB_Day > 0)
{
    text = "0";          // 소아 접미사 제거
    M_Is6Age = false;
}
```

#### 산정코드 접미사

| 대상 코드 | 접미사 | 예시 |
|-----------|--------|------|
| Z2000 (조제기본료) | `600` (의미I=6) | Z2000600 |
| Z2000 + 야간 | `610` | Z2000610 (6세+야간) |
| Z2000 + 공휴 | `650` | Z2000650 (6세+공휴) |

> **주의:** Z3000(복약지도료)에는 6세 미만 단독 코드(Z3000600)가 **없다**. 6세+야간=Z3000010, 6세+공휴=Z3000050, 6세 단독=Z3000(기본) 사용. (비즈팜 확인)

#### 금액 공식

```
최종점수 = 기본점수 + 소아가산점수
금액 = (기준금액 × 최종점수).To원단위사사오입()
```

- 처방조제: P소아가산점수
- 직접조제: D소아가산점수
- 소아 나이 기준: ST조제료적용일.소아나이 (환경설정 참조)

#### 중복 가능 범위

소아 가산은 아래 가산과 **중복 적용** 가능:
- 야간/공휴 가산 + 소아 가산 (비즈팜: Holiday_gb "7"/"8")
- 토요 가산 + 소아 가산
- 소아심야 가산 + 소아 가산 (해당 시)

단, **가루약 가산 시에는 소아 가산 미적용** (Check소아가산_조제료 로직).

#### 금액 공식 -- 분리 계산

```
토요 가산은 기본금액과 가산금액을 분리 계산 후 합산:

토요가산점수 = Math.Round(기본점수 × 토요일가산률, 2)
기본점수 = Math.Round(기본점수, 2)

최종금액 = (기준금액 × 기본점수).To원단위사사오입()
         + (기준금액 × 토요가산점수).To원단위사사오입()
```

유팜 소스 확인 (Base조제료Logic.cs 131~134줄):

```csharp
else if (토요가산여부)
{

    num2 = Math.Round(num2, 2);                         // 기본 점수

// 최종: (기준금액 × num2).사사오입 + (기준금액 × num).사사오입
```

### 4-6. 가루약 가산

### 4-8. 차등 가산

#### 개요

차등수가는 약국의 **영업시간** 기반으로 산정되며, 정상 영업시간 내 조제 시 차등수가를 적용하지 않는다(`text3 = "1"`). 이는 약국관리료(Z1000)의 의미III에 반영된다.

#### EDB 소스 상세 (PrsBillCalcM.SugaCalc.cs, 라인 1453~1547)

```
1. 영업시간 조회: SelectManageInOutTime(dosDate)
   → inTime, outTime 산출
   → num23 = outTime - inTime (영업시간 길이)

2. 기본값: m_isChadung_Yn = true (차등 해당)

3. 2017.10.01 이후:
   - 평일: num23/100 >= 8 AND (야간 OR 심야) → 차등 미적용
   - 토요: num23/100 >= 4 AND (야간 OR 심야) → 차등 미적용
   - 일요: 2015.12.01 이후 AND M_Grade_Sat_In="N" → 차등 미적용

4. 2017.10.01 이전:
   - 평일: num23/100 >= 8 → (영업시작+9시간) 범위 외 AND (야간 OR 심야) → 차등 미적용
   - 토요: num23/100 >= 4 → (영업시작+4시간) 범위 외 AND (야간 OR 심야) → 차등 미적용

5. 약물안전서비스 Y/A → 차등 미적용
6. 2015.12.01 이후 AND M_Grade_Sat_In="N" AND 공휴일 → 차등 미적용
7. 보험코드 첫자리 != "C" → text3 = "0" (비해당)
8. 2015.12.01 이후 AND M_Grade_Sat_In="N" AND (심야/야간/공휴) AND (공휴 OR 토/일) → text3 = "0"
```

#### 접미사 결정

```
차등 미적용(m_isChadung_Yn = false): text3 = "1"
비해당(보험코드 != "C"): text3 = "0"
차등 해당: text3 = "0" (기본)
```

#### 최종 접미사 조합 (EDB, 라인 1548)

```csharp
stringBuilder.Append(text + text2 + text3);
// text  = 연령 접미사 ("6" = 6세미만, "0" = 기타)
// text2 = 시간 접미사 ("1"=야간, "2"=심야, "5"=공휴, "3"=토요)
// text3 = 차등 접미사 ("1"=차등미적용, "0"=기타)
```

### 6-2. 불일치 사항

#### (1) Z3000 복약지도료의 6세 미만 처리

| 소스 | 처리 |
|------|------|
| **비즈팜** | Holiday_gb="6" → Z3000 (기본). 6세 단독 코드 없음 |
| **유팜** | 소아가산여부 파라미터로 소아가산점수 추가 |
| **EDB** | text="6" 접미사로 코드 생성 |
| **공단** | Z30(복약지도료)에 별도 산정코드 미기재 |

**검증:** 비즈팜의 Z3000에서는 6세 미만 단독 가산 코드가 없고 기본 Z3000을 사용한다. 이는 Z3000의 산정코드 체계에 의미I가 정의되어 있지 않기 때문이다. 유팜은 소아가산점수를 점수 단위에서 더하는 방식이므로 코드가 아닌 점수로 가산을 반영한다. **방식은 다르나 결과는 동일** (기본점수 + 소아가산점수).

#### (2) 소아심야 2023.11.01 분기

| 소스 | 처리 |
|------|------|
| **유팜** | `소아심야산정코드변경시작일` 기준으로 기본조제기술료/복약지도료에만 `소아야간가산률_기본` 적용 |

| 소스 | 우선순위 |

## 7. 주의사항

### 7-1. 구현 시 주의

1. **가산 판정 순서 엄수**: if-else 체인에서 가루약을 반드시 최상위로 배치. 순서 오류 시 야간+가루약 동시 해당 케이스에서 이중 가산 발생.

2. **토요 가산 분리 라운딩**: 토요 가산만 기본/가산 별도 라운딩 후 합산. 단일 라운딩으로 처리하면 수 원 차이 발생.

3. **소아심야 2023.11.01 분기**: 기본조제기술료와 복약지도료에만 `소아야간가산률_기본`을 적용하고, 나머지 항목은 기존 `소아야간가산률`을 유지해야 한다.

4. **비대면 플래그 흡수**: 비대면 시 야간/공휴/심야 플래그를 false로 전환한 후 ZC코드를 설정해야 한다. 플래그만 흡수하고 ZC코드를 미설정하면 가산이 누락된다.

5. **차등가산 text3과 보험코드**: 보험코드 첫 자리가 "C"가 아니면 차등가산 자체가 비해당(`text3 = "0"`). 차등수가는 건강보험 외래 처방조제에만 적용된다.

## 부록 B: 차등수가 산출 공식 상세

### B-1. 1일 평균 조제횟수 산출

```
1일_평균_조제횟수(n) = 주(또는 월)_총_조제횟수 ÷ 약사별_총_조제일수
→ 소수점 첫째 자리에서 절사 (정수로 만듦)
```

- 약사별 누적 조제횟수/조제일수는 외부 데이터 연동 필수 (단일 명세서로는 판정 불가)
- 원문 PDF p.293(=p.295), p.467~468 참조

### B-2. 구간별 가중치 (2026 상대가치점수표 확인 완료)

| 구간 | 1일 평균 조제횟수 | 가중치 |
|------|------------------|--------|
| 1구간 | 75건 이하 | 1.00 (100%) |
| 2구간 | 76~100건 | 0.90 (90%) |
| 3구간 | 101~150건 | 0.75 (75%) |
| 4구간 | 151건 이상 | 0.50 (50%) |

> 출처: 상대가치점수표 제1부 일반원칙 III (고시 제2025-186호)

### B-3. 차등지수 산출 공식

```
n ≤ 75:  차등지수 = 1.0 (감액 없음)

75 < n ≤ 100:
  차등지수 = {75×1.00 + (n-75)×0.90} / n

100 < n ≤ 150:
  차등지수 = {75×1.00 + 25×0.90 + (n-100)×0.75} / n

150 < n:
  차등지수 = {75×1.00 + 25×0.90 + 50×0.75 + (n-150)×0.50} / n

→ 소수점 여덟째 자리에서 4사5입 → 소수점 일곱째 자리까지 유지
```

**예시 (n=120):**
```
차등지수 = {75×1.00 + 25×0.90 + 20×0.75} / 120
         = {75 + 22.5 + 15} / 120
         = 112.5 / 120
         = 0.9375000
```

### B-4. 차등수가 청구액 산출

```
각 명세서별:
  조제료등_합계 = 조제료(Z40~Z43) + 약국관리료(Z10) + 조제기본료(Z20) + 복약지도료(Z30)
                  (단, Z10 의미I='1' 차등수가 제외분은 합계에서 제외)
  차감액 = 조제료등_합계 × (1 - 차등지수)
  차등수가청구액 = 청구액 - 차감액

전체:
  차등수가청구액_합계 = Σ(각 명세서별 차등수가청구액)
```

> 차등수가는 심사청구서 수준에서 합산 기재하며, 개별 명세서의 요양급여비용총액1이나 본인일부부담금 자체를 변경하지 않는다. 청구서 제출 시 차등지수, 차등수가청구액 필드에 기재.
>
> 차등수가는 건강보험(보험코드 첫 자리 "C") 외래 처방조제에만 적용된다.
