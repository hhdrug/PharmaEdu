## 1. 개요

약국에서 보험처방전에 따라 조제한 약제비의 본인부담금은 **보험유형(건강보험/의료급여/보훈/자동차보험/산재)** 에 따라 산출 방식이 완전히 다르다. 본 문서는 4개 소스 시스템의 코드를 교차 검증하여 통합 규칙을 정리한다.

### 1.1 계산 흐름 요약

```
총약제비 = 약가(01항) + 조제료(02항)
요양급여비용총액1 = trunc10(총약제비)          ← 10원 미만 절사
본인부담_기준액 = 요양급여비용총액1 - 장려금    ← 대체조제/사용장려금 제외
본인일부부담금 = f(보험유형, 기준액, 나이, 특정기호, ...)
청구액 = 요양급여비용총액1 - 본인일부부담금
```

### 1.2 용어 정의

| 용어 | 소스별 변수명 | 의미 |
|------|-------------|------|
| 요양급여비용총액1 | Insur_Amt(비즈팜), PbSum(EDB), 총약제비(유팜) | 급여 약가+조제료의 10원 절사 합계 |
| 본인일부부담금 | Insur_Self_Amt(비즈팜), PbUserPrice(EDB), 본인부담금(유팜) | 환자가 부담하는 금액 |
| 정액/정률 구분 | Percent_GB(비즈팜), m_Fixed(EDB), 정액여부(유팜) | 고정금액 vs 비율 적용 구분 |
| 장려금 | 저가대체약가장려금(유팜) | 본인부담 산정 전 기준액에서 제외 |

## 2. 총약제비 산출 -- 약가+조제료 -> 총액(10원 절사)

```
총약제비 = 급여약가합계 + 급여조제료합계
요양급여비용총액1 = Floor(총약제비 / 10) × 10
```

**소스별 확인:**
- **비즈팜**: `Insur_Amt = Int(Tot_Drug_Amt / 10) * 10` (라인 9583)
- **EDB**: `m_prsBillM.PbSum = Math.Truncate((...) / 10m) * 10m` (라인 3459)
- **유팜**: `src.To십원미만내림()` (Base처방총약제비Logic.cs:107)
- **공단**: "요양급여비용총액 1" — p.662~663 예시 일치

> **4소스 일치**: 모두 10원 미만 절사(Floor). 불일치 없음.

### 3.1 일반 건강보험 (C10) -- 30%

```
본인일부부담금 = trunc100(본인부담_기준액 × 30%)
```

**소스별 확인:**
- **비즈팜**: `Insur_Self_Amt = Int(Insur_Amt * 30 / 100)` → 100원 절사 (라인 9651)
- **EDB**: `PbUserPrice = Truncate(PbUserPrice * (m_insuRate/100) / 100) * 100` (라인 3600). m_insuRate=30
- **유팜**: `(cur총약제비 * calcResult.본인부담율).To백원미만내림()` — 본인부담율 0.3 (P정률)
- **공단**: p.663 "요양급여비용총액 1의 30%를 본인이 부담" + "100원 미만 절사"
- **docs**: "부담률 = 30%, 본인일부부담금 = trunc100(본인부담_기준액 × 30%)"

> **4소스+공단 일치**: 30%, 100원 미만 절사.

### 3.2 65세 이상 3구간 (2018-01-01 시행)

| 구간 | 조건 | 본인부담 | 절사 |
|------|------|---------|------|
| 1구간 | 총약제비 <= 10,000원, 산정특례 없음 | **정액 1,000원** | - |
| 2구간 | 10,000원 < 총약제비 <= 12,000원 | **20%** | 100원 절사 |
| 3구간 | 총약제비 > 12,000원 | **30%** | 100원 절사 |

> **검증 노트 (CH05 점검 2026-04-08):**
> - **약국 65세 정액 = 1,000원** (10,000원 이하 구간). 2018.01.01 시행 이후 유지.
> - **EDB Mock C10/C11/C20/C21 FixCost=1,500은 채택하지 않음.** EDB Mock 값은 의원의 1,500원 기준을 잘못 가져온 것으로 추정. 약국은 공단원문(`docs/약국_약제비_계산_로직.md:540`) "1만원 이하: 1,000원, 종전 1,200원에서 인하"에 따라 **1,000원**이 정답.
> - 출처: 데일리팜·약사공론·보건복지부 고시 2025-186호·HIRA 본인부담기준 안내 (웹 4출처 교차검증, CH00 §4.2 참조)
> - **CH00 §4.2와 일관성 유지.** 둘 다 1,000원으로 통일됨.

**소스별 확인:**

- **비즈팜**: intZBase(DB "ZBase" 조회, 기본값 8000→현행 10000), intZBaseAmt(DB "ZBaseold" 조회, 기본값 1000). Percent_GB 결정 후 정액/정률 분기 (라인 9614~9632). 2구간(12,000원)은 비즈팜에서 **직접 구현되어 있지 않음** — DB 마스터(PZCVALUE)에 의존하며, 2018년 고시 반영 여부는 DB 데이터에 좌우됨.
- **EDB**: `m_Age >= 65 && PbSum - Extype9 <= 10000` → FixCost 적용 (라인 3484~3527). 12,000원 구간은 `Age65_12000Less` 필드로 요율 변경 (라인 3551~3553). m_insuRate를 Age65_12000Less 값으로 교체.
- **유팜**: `Calculate본인부담금_노인()` — P약제비기준(10,000원), P약제비기준2(12,000원), P정액노인(1,000원), P정률노인(0.20) 사용. 기준가 초과 시 -1 반환하여 일반 정률 적용.
- **공단**: p.663 "65세 이상이면서 요양급여비용총액 1이 10,000원 이하일 경우에는 정액 본인일부부담금(1,000원)을 적용"

> **소스간 차이**: 비즈팜은 2구간(20%)을 코드에 하드코딩하지 않고 DB에 의존. EDB/유팜은 12,000원 경계를 명시적으로 처리. **결과값은 일치**.

**정액 적용 시 총약제비 < 정액금액인 경우:**
- **EDB**: `num < FixCost → PbUserPrice = num` (총약제비 자체가 본인부담, 라인 3506~3509)
- **유팜**: `P정액노인 > 총약제비 → 총약제비.To백원미만내림()` (라인 66~69)
- **비즈팜**: 해당 로직 미발견 (정액금액을 그대로 적용 — 총약제비가 1,000원 미만일 때 차이 발생 가능)

### 3.3 6세 미만 소아 (SixAgeRate)

```
부담률 = 30% × 70% = 21%
본인일부부담금 = trunc100(본인부담_기준액 × 21%)
```

**소스별 확인:**
- **비즈팜**: `CalSelfAmt()` — `Insur_Self_Amt = CLng((Insur_Self_Amt * 7/10) / 100 - 0.5) * 100`. 먼저 30%로 계산 후 70% 곱하여 100원 절사 (라인 19238~19257). Holiday_gb가 6/7/8인 경우에만 호출.
- **EDB**: `num2 *= insuRateM.SixAgeRateDecimal / 100m` (Helpers.cs:196). SixAgeRate를 m_insuRate에 직접 곱함 → 요율 자체가 21%로 변환.
- **유팜**: `result *= _보험기준가.소아정률` (Base처방총약제비Logic.cs:125). 소아정률=0.7로 30%×0.7=21%.
- **공단**: p.656 "6세 미만의 경우에는 본인이 부담할 비용의 부담률의 100분의 70에 해당되는 비용을 부담"
- **docs**: "6세미만(1세미만 포함): 성인 부담의 70% → 30% × 70% = 21%"

> **주의**: 약국은 의료기관과 달리 "1세 미만" 별도 경감이 없음. 6세 미만으로 통합 (docs 확인).

> **비즈팜 구현 특이점**: CalSelfAmt()는 Holiday_gb=6/7/8(6세미만 관련 코드)일 때만 호출되며, 일반 6세미만은 Holiday_gb="6"으로 설정된 상태에서 Pres_Bill() 후단에서 처리. 다른 소스는 부담률 자체를 21%로 설정하여 계산.

> **비즈팜 한계**: V252(50%) 등 제6조 특례는 미구현. chkSan=1(20%)로 통합 처리하여 **50%/40% 케이스에서 과소 청구** 가능성.

### 3.6 공상 (공무상 재해 — 별도 플래그)

```
본인일부부담금 = 0   // 전액 공단/공무원연금공단 부담
```

> **개념 정리:** "공상(公傷)"은 보험코드가 아니라 **환자 상태 플래그**이다. 공무원이 공무 수행 중 다치거나 병에 걸린 경우 공무원연금공단이 처리하며, 약국 본인부담은 0원이 된다. 어떤 건강보험 코드(C10/C11/C20/C21 등)를 가진 환자든 "이번 진료는 공상"으로 마킹되면 본인부담 0원이 강제 적용된다.

**소스별 처리:**
- **비즈팜**: `chkGong=1` 체크박스 플래그 → `Insur_Self_Amt = 0`
- **EDB**: 별도 분기 미확인 (실DB 운영 시 InsuCode 또는 sbrdnType으로 처리 추정)
- **유팜**: 별도 클래스 또는 면제 플래그로 처리
- **공단원문**: "공무원연금법 또는 사립학교교직원 연금법에 따른 공무상 요양" — 시행령 별표2 본인부담 면제 항목

> **CH05 점검 2026-04-08 정정:** 사양 초판은 본 항목을 "공상 (C21)"로 표기하여 C21 보험코드와 매핑했으나, EDB Mock(`MockCostCalcRepository.cs:400`)에서 **C21은 "지역가입자 세대주"**임이 확인됨. 공상은 보험코드와 무관한 환자 상태 플래그이므로 매핑 분리.

### 5.2 보훈 본인부담금 기본 공식

```
보훈청구액      = Trunc10(요양급여비용총액1 × 감면율)   ← 끝수 안전판 (CH07 결정)
본인부담_기준액 = 요양급여비용총액1 - 보훈청구액
본인일부부담금  = Trunc100(본인부담_기준액 × 본인부담률)
```

여기서 `본인부담률`은 일반 30%이지만, 산정특례 시 해당 특례율(5%/10% 등)을 적용.

**소스별 확인:**
- **비즈팜(조합기호 있음, 일반, 정률)**: `60%면제 → Int(Insur_Amt * 12/100)` = 30%×40% = 12%. `50%면제 → 15%`, `30%면제 → 21%` (라인 9769~9775). 역산 방식이 아닌 합산율 직접 적용.
- **EDB**: `PbMpvaPrice` (보훈국비청구액) 산출 후 `PbUserPrice = (num - PbMpvaPrice) × (insuRate/100)` 방식으로 100원 절사 (`InsuRateCalc2.cs:58-79, 3557-3600`). **보훈청구액은 `Math.Truncate(num × ((100m - bohunRate) / 100m) / 10m) * 10m` = Trunc10**.
- **유팜**: `보훈감면Base.Calculate보훈본인부담금()` — `num3 = (총약제비 - 장려금 - 보훈약가) × (1-감면율) × 본인부담율` (`보훈감면Base.cs:19-43`). **보훈청구액은 `Math.Ceiling((총약제비 - num) / 10m) * 10m` = Ceil10**.
- **docs**: `본인일부부담금 = trunc100(본인부담_기준액_보훈 × 30%)`

> **검증 노트 (CH05 점검 2026-04-08, CH07 결정 반영):**
> - **보훈청구액 끝수처리 = Trunc10 채택** (EDB 일치). 유팜은 Ceil10이지만 우리는 EDB 방식 채택.
> - 결정 근거: 국고금관리법 제47조(국가는 10원 미만 끝수 지급 X) + HIRA·보훈공단 심사시스템이 10원 단위 입력만 수신 + 절사 방향이라 삭감 리스크 0.
> - 상세: CH07 §6-6 참조.

### 5.7 보훈 + 조합기호 유무에 따른 분기 (비즈팜 고유)

비즈팜은 보훈감면 환자에서 **조합기호(txtAsso_id/txtInsurance_id) 유무**에 따라 다른 부담률을 적용:

| 감면등급 | 조합 있음 | 조합 없음 |
|---------|----------|----------|
| 60%면제 | 12% | 40% |
| 50%면제 | 15% | 50% |
| 30%면제 | 21% | 70% |

- **조합 있음**: `30% × (100%-감면율)` = 보험 부담률 × 잔여율
- **조합 없음**: `(100%-감면율)` = 잔여율 전액 본인부담 (건보 미적용)

> **비즈팜 고유 로직**: EDB/유팜에는 "조합기호 유무" 분기가 없음. EDB는 보험코드(G10 등)로 이미 보훈 대상을 판별하므로 조합 유무와 무관하게 동일 계산.

**docs 확인**: "보훈감면 환자가 65세 이상이고 약제비 1만원 미만이더라도, 65세 정액 1,000원 규칙은 적용되지 않는다."

- **유팜**: `보훈감면Base.Calculate보훈본인부담금()` — 65세 정액 분기에서 `P정액노인 > num3(보훈감면후)` 조건으로 보훈감면 후 금액 기준 판단. 감면 후 금액이 정액보다 작으면 감면후 금액을 그대로 사용 (보훈감면Base.cs:31~41).
- **EDB**: 65세+10,000원 이하 분기 내에서 보훈코드 존재 시 FixCost 대신 보훈 로직 우선 (라인 3518~3527).

### 5.9 비즈팜 보훈 50%면제+암환자 BUG

비즈팜 라인 9732에서 발견:
```vb
Insur_Self_Amt = Insur_Self_Amt = Int((Int(Insur_Amt * 50/100 * 0.1) * 10) * 10/100)
```

`=`가 두 번 사용되어 VB6에서 `A = (A = B)` 비교로 해석됨. **50%면제+암환자+조합기호 있음** 조합에서 본인부담금이 0 또는 -1이 되는 버그.

> **영향**: 극히 드문 케이스(보훈 50%면제+암환자+조합)이나, 해당 조건에서 비즈팜은 오류 금액 산출.

## 8. 전액본인부담(U항) + 비급여(W항) 혼재 시 조제료 귀속

### 8.1 U항(100/100 본인부담) 처리

```
100/100본인부담금총액 = trunc10(U항_합계)
요양급여비용총액2 = trunc10(요양급여비용총액1 + 100/100본인부담금총액)
```

U항 약제만 조제한 경우에도 조제료는 요양급여비용으로 산정하여 일수 산입 (docs 확인, p.469).

### 8.2 비급여(W항) 조제료

비보험 약품만 있는 경우 조제료는 **비급여 조제료**로 산정.
보험+비보험 혼재 시: 급여 조제료는 보험 약품 기준, 비급여 조제료는 전체 약품 기준에서 급여 조제료를 차감한 금액.

**비즈팜**: `chkBiboPres=1` → 2차 `Pres_Bill(True)` 호출로 비보험 조제료 재계산 (라인 8984~9028).
**유팜**: `Calculate비급여조제료(조제매출, 급여조제료)` — 급여 조제료를 전체에서 차감.

### 8.3 선별급여(A/B/D/E항) 본인부담

```
100/100미만_본인일부부담금 = trunc10(A항×50% + B항×80% + D항×30% + E항×90%)
```

**유팜**: `백대백미만본인부담금 = (A×0.5 + B×0.8 + D×0.3 + E×0.9).To십원미만내림()` (Base총약제비Logic.cs:145).

> 선별급여는 요양급여비용총액1과 **독립**적으로 계산됨 (docs 확인).

### 9.2 소스별 절사 구현

**비즈팜 (Pres_Bill 후처리, 라인 9798~9820):**
```
IF 2007.08.01 이후 AND 정률:
    IF 자동차보험 또는 보훈비급여: 10원 절사
    ELIF 6세미만: CalSelfAmt() → 100원 절사 (70% 감면 포함)
    ELSE: 100원 절사
ELIF 2007.08.01 이후 AND 정액:
    IF 자동차보험/보훈비급여: 처리없음
    ELSE: 100원 절사
ELSE (2007.08.01 이전):
    10원 절사
```

**EDB:**
- C타입 일반: `trunc(... / 100) * 100` — 100원 절사 (라인 3600)
- C타입 100%: `trunc(... / 10) * 10` — 10원 절사 (라인 3602)
- C31/C32 (2016.09.29 이후): 100원 절사 (라인 3563)
- C31/C32 (2016.09.29 이전): 10원 절사 (라인 3567)
- D타입 B014: `trunc(... / 10) * 10` — **10원 절사** (라인 3694)
- D타입 정액: 절사 없음 (고정금액)
- G타입 100%: 10원 절사, <100%: 100원 절사 (라인 3801, 3805)

**유팜:**
- `To백원미만내림()` = 100원 절사
- `To십원미만내림()` = 10원 절사
- 건강보험: 100원 절사 (Base처방총약제비Logic.cs:37)
- 보훈감면30/50/60: 10원 절사 (보훈감면Base.cs:39 조건부)

> **docs 종합 (§7-5)**: 건강보험 외래/약국 = 100원 절사, 의료급여 약국 = 10원 절사, 청구액 = 절사 없음. 자세한 사항은 CH07 참조.

### 9.3 비즈팜 gSelfAmt (수납금액 절사 옵션)

비즈팜은 총수납금액(pnl_TotalSelfAmt) 표시 시 약국별 설정으로 추가 절사:

| 설정값 | 방식 | 수식 |
|--------|------|------|
| 0 | 10원 절사 | `Int(금액 * 0.1) * 10` |
| 1 | 100원 절사 | `Int(금액 * 0.01) * 100` |
| 2 | 100원 반올림 | `Int(금액 * 0.01 + 0.5) * 100` |

> 이는 **수납 표시용**이며 심사청구금액과는 무관. EDB/유팜에도 유사한 환경설정이 있음.

건강보험에서 V252=50%, V352=40%인 것과 달리, **의료급여 수급권자**의 경증질환 약제비는 상급종합/종합병원 구분 없이 일률 **3%** 부담이다.

**기본 공식:**
```
본인일부부담금 = MAX(trunc10(본인부담_기준액 × 3%), 500)
```

- 3%가 500원 미만이면 **최소 500원** 적용
- **절사**: trunc10 (10원 미만 절사) — 건강보험 차상위의 trunc100(백원 절사)과 다름
- 건강생활유지비 지급 대상자는 해당 본인부담금이 건생비에서 차감됨

**적용 판정표:**

| 대상 | V252 시 3% 적용 여부 | 비고 |
|------|---------------------|------|
| 보호2종 | **O** — 3% 부담(trunc10), 최소 500원 | |
| 보호1종 건강생활유지비 대상 | **O** — 3% 부담, 건생비에서 차감 | |
| 보호1종 B코드(B005 등) | **O** — 비선택기관 3% 부담 | MT018 확인 필요 |
| 보호1종 그 외 | **X** — V252 있어도 3% 미적용 | 기존 1종 정액(500원) 유지 |
| 본인부담 면제 대상자 | **X** — V252 있어도 면제 유지 | §12.4 면제 7종 참조 |

**적용 제외 사유** (HIRA 안내서 p.35 Q59):
- 의료급여 본인부담 면제자
- 선택의료급여기관(종합병원급 이상)에서 처방받는 경우
- 당뇨병으로 인슐린 투여 중인 경우
- 일부 상병에서 6세 미만 소아
- 의원 진료의뢰서 지참 시 종합병원 진료 (90일 한도)

**근거:** HIRA 「2025 알기 쉬운 의료급여제도」 p.34, docs/약국_약제비_계산_로직.md §9-10

### 12.2 보충 B: 차상위 2종(E/F) 세부 규칙

본문 §3.4에서 차상위 2종의 기본 구조를 기술했으나, 아래 세부 분기가 누락되어 있었다.

**(1) f028 자립청소년 — 14% 정률**
```
IF 처방전_특정기호에 'f028' 포함:
    본인일부부담금 = trunc100(본인부담_기준액 × 14%)
```
자립청소년은 차상위 2종이지만 정액 500원이 아닌 **14% 정률**을 적용한다.

**(2) V252/V352/V452 경증질환 — 3%(최소 500원)**
```
IF 처방전_특정기호 IN ('v252', 'v352', 'v452'):
    본인일부부담금 = MAX(trunc100(본인부담_기준액 × 3%), 500)
```
차상위 2종도 경증질환 시 3% 차등제가 적용된다. 절사는 trunc100 (건강보험 차상위 기준, 로직.md §7-1).

**(3) 보건기관 처방전 — 0원**
보건소/보건지소/보건진료소에서 발행한 처방전인 경우 본인부담 0원 (별표2 라목 약국란).

**(4) F코드(장애인) — 장애인의료비 750원 차감**
```
// F 대상자: 의원급 외래 장애인의료비 750원을 본인부담금에서 차감
환자_실부담 = 본인일부부담금 - 장애인의료비(750)
청구액 = 요양급여비용총액1 - 본인일부부담금 - 장애인의료비
```
F코드 대상자는 본인부담금(500원)에서 장애인의료비를 차감하므로 실부담이 줄어든다 (p.611).

**분기 우선순위 (차상위 2종 내):**
```
f028(자립청소년 14%) > V252/V352/V452(3%, 최소500원) > 보건기관(0원) > 직접조제(900원) > 처방조제(500원)
```

**근거:** 국민건강보험법 시행령 별표2 제3호 라목, docs/약국_약제비_계산_로직.md §7-1

### 12.3 보충 C: 보훈위탁 국비환자 상세 (공상등구분 4/7)

본문 §5에서는 보훈감면(30%/50%/60%/90%/100%)을 다루었으나, **보훈위탁 국비환자**(공상등구분 4/7)의 상세 규칙이 누락되어 있었다.

**(1) 기본 구조**

| 대상 | 국비질환 | 타(일반)질환 |
|------|---------|-------------|
| 국가유공자 1~7급, 고엽제 등급 | 전액 국비 | 본인부담의 90% 국비지원 |
| 경상이자 | 전액 국비 | 국비 대상 아님 |
| 무자격자 | 전액 국비 | 전액 국비 또는 90% |

- **전액 국비**: 보험급여 등재 의약품(100/100 포함) + 비급여(건위소화제만) + 기술료
- **일부본인부담대상 전상군경(MT038='2')**: 국비지원비용의 **10%** 본인부담 (90% 지원)

**(2) MT038 코드별 처리**

```
IF 국비질환_조제:
    청구액 = 0  // 전액 보훈부담
ELSE IF MT038 == '2':  // 일부본인부담대상 전상군경, 타질환
    보훈국비지원비용 = 요양급여비용총액1 × 90%
    본인부담 = 요양급여비용총액1 - 보훈국비지원비용  // 10%
ELSE:
    청구액 = 0  // 전액 보훈부담 (무자격자 등)
```

**(3) MT038='A' 도서벽지 약국 감면 코드전환**

약국에서 보훈위탁 60% 감면환자가 도서벽지에 소재하는 경우:
- 공상등구분 `6`, MT038 `'A'` 기재
- 도서벽지 외: 공상등구분 `0(E)`, MT038 미기재

**(4) 약국 국비환자 코드 전환 규칙 (2013.01.01~)**

일부본인부담대상 전상군경의 타(일반)질환 조제 시:
- 보험자종별구분: 7 → 4(5)
- 공상등구분: 7 → 4

**근거:** PDF p.585, p.587, p.599, docs/약국_약제비_계산_로직.md §7-4(b)

### 12.4 보충 D: 의료급여 1종 면제 대상 8종

의료급여 1종 수급권자 중 아래 대상은 약국 본인부담금(500원)도 **전액 면제**된다.

| # | 면제 대상 | 근거 |
|---|----------|------|
| 1 | **18세 미만** | 의료급여법 시행령 제13조, 별표1 |
| 2 | **20세 미만 중·고등학교 재학생** | 시행규칙 제19조의4 |
| 3 | **임산부** | 시행령 별표1 |
| 4 | **가정간호 대상자** | 시행령 별표1 |
| 5 | **선택의료급여기관 이용자** | 시행령 별표1 |
| 6 | **행려환자 / 노숙인 진료시설 이용 노숙인** | 시행령 별표1 |

**구현 시 유의:**
- 면제 대상은 의료급여종별(1종) + 수급권자 속성으로 판정
- 면제 대상이면 본인부담금 = 0원
- V252(경증질환 3%)가 있어도 면제 대상자는 **면제 유지** (§12.1 적용 판정표 참조)

**근거:** 의료급여법 시행령 제13조, 별표1, 시행규칙 제19조의4, docs/약국_약제비_계산_로직.md §9-10
