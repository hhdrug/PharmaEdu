# CH03. 조제료 수가 계산 로직 통합

> **목적**: 다른 AI가 이 문서만 읽고 조제료 수가 계산 코드를 작성할 수 있도록 정리한다.
>
> **소스**: 비즈팜 VB6(Ppre1000__.frm), 유팜 C#(Base조제료Logic.cs + 처방/직접 서브클래스), EDB C#(PrsBillCalcM.SugaCalc.cs), 공단 문서(원문PDF 기준 구현명세), docs/ 기존 분석 2건
>
> **범위**: 조제료 항목별 금액 산출 로직. 본인부담금/절사 규칙은 CH05/CH07에서 다룬다.

## 1. 개요 -- 조제료 계산의 위치와 역할

### 1-1. 약제비 내에서의 위치

```
약제비 = 약가(01항) + 조제료(02항)

조제료 = 기본조제료 + 약품조제료 + 특수조제료

기본조제료 = 약국관리료(Z1000)
           + 기본조제기술료(Z2000)
           + 복약지도료(Z3000)
           + 의약품관리료(Z5000/Z5001/Z5011)

약품조제료 = 내복조제료(Z4xxx)
           + 외용조제료(Z4120/Z4220)
           + 내외용동시조제료(Z4121/Z4221)
           + 주사조제료(Z4130)
           + 내복조제료_가루약(Z4010/Z4020)

특수조제료 = 야간조제관리료(Z7001)
           + 비대면조제관리료(ZC001~ZC004)
           + 코로나19관련료(ZH001~ZH004)
           + 명절조제지원금(ZE100)
```

### 1-2. 전체 계산 파이프라인에서의 위치

```
[STEP 1] Clear_Amt() -- 금액 변수 초기화
[STEP 2] Check_Holiday() -- 야간/공휴/6세미만 판정 → Holiday_gb 결정
[STEP 3] Check_Drug() -- 약품 분류(내복/외용/주사), 약가 합산, 투약일수 결정
[STEP 4] ★ Pres_Bill() -- 조제료 수가 계산 (본 문서의 핵심)
[STEP 5] SelfRate() -- 본인부담률 결정 (CH05 범위)
[STEP 6] 본인부담금/청구액 산출 (CH05 범위)
```

### 1-3. 산정 전제 조건

조제료 계산에 진입하기 전에 다음이 결정되어 있어야 한다:

| 입력 | 설명 | 결정 시점 |
|------|------|-----------|
| `presDosage` (투약일수) | 급여 내복약 최대 투약일수 | STEP 3 |
| `rIn_Chk` / `rOut_Chk` | 내복약/외용약/주사제 존재 플래그 | STEP 3 |
| `Holiday_gb` | 야간/공휴/6세미만 구분 | STEP 2 |
| `chkDate` | 처방일 >= 2000.09.01 여부 | STEP 4 진입 시 |
| `presDate` | 처방(조제)일자 | 입력 |
| 처방조제/직접조제 구분 | H024/H124 vs H025/H125 | 입력 |

## 2. 계산 순서 -- 9개 수가항목 산정 순서

모든 소스(비즈팜/유팜/EDB)에서 아래 순서를 따른다.

```
[순서 1] Z1000 약국관리료
[순서 2] Z2000 기본조제기술료 (+ 토요가산)
[순서 3] Z3000 복약지도료 (+ 토요가산)
[순서 4] Z4xxx 약품조제료 -- 내복/외용/내외용동시/주사 (+ 토요가산, + 가루약가산)
[순서 5] Z5000 의약품관리료 (기본 + 일수별)
[순서 6] Z7001 야간조제관리료 (달빛어린이, 처방조제만)
[순서 7] ZH/ZC 코로나19/비대면 조제관리료
[순서 8] ZE100 명절조제지원금
[순서 9] 합산 → Pres_Amt (총 조제료)
```

> **주사제만 처방인 경우**: 순서 1~4를 건너뛰고 Z5000/Z5010만 적용 후 종료한다 (비즈팜 라인 9321~9339). 유팜/EDB도 동일하게 주사제만일 때 기본조제기술료/복약지도료/약국관리료를 0원 처리한다.

## 3. 각 항목별 계산 공식

### 3-1. Z1000 약국관리료

**산정단위**: 방문당 1회

**수가조회 방식 비교**:

| 소스 | 방식 |
|------|------|
| 비즈팜 | `LoadRsZCode("Z1000", presDate)` → PZCVALUE 테이블에서 `insurance_amt` 조회 |
| 유팜 | `Get조제료Item(조제료CodeTypes.약국관리료, 최대내복조제일수)` → `기준금액 × 점수` 계산 |
| EDB | `mediWageM.GetPrice("Z1000접미사")` → MediWageM 딕셔너리에서 단가 직접 조회 |

**금액 산출 공식**:

```
# 비즈팜
ChkDosage = (in_Gb == "1") ? 1 : presDosage
Z1000_금액 = ChkDosage × insurance_amt

# 유팜
Z1000_금액 = (기준금액 × 기본점수).To원단위사사오입()

# EDB
Z1000_금액 = (int)mediWageM.GetPrice(코드)
```

**가산 적용**: 야간/공휴일/6세미만 가산 **없음** (단일 코드)

**차등수가 접미사** (EDB):
- 야간(`text2="1"`) 또는 심야(`text2="2"`) → `Z1000 + "00" + text3` (text3: 차등가산코드)
- 그 외 → `Z1000` 그대로

**의사코드**:

```python
def calc_Z1000(presDate, presDosage):
    row = load_z_code("Z1000", presDate)
    if row is None:
        return 0
    multiplier = 1 if row.in_Gb == "1" else presDosage
    return multiplier * row.insurance_amt
```

### 3-2. Z2000 기본조제기술료

**산정단위**: 방문당 1회

**Z코드 매핑 테이블** (처방일 >= 2000.09.01, `chkDate=True`):

| Holiday_gb | Z코드 | 의미 |
|------------|-------|------|
| `N`/`0`/기타 | Z2000 | 기본 |
| `1` (야간) | Z2000010 | 야간 가산 |
| `5` (공휴일) | Z2000050 | 공휴일 가산 |
| `6` (6세미만) | Z2000600 | 6세미만 가산 |
| `7` (6세+공휴) | Z2000650 | 6세미만+공휴일 |
| `8` (6세+야간) | Z2000610 | 6세미만+야간 |

> `chkDate=False` (2000.09.01 이전): 공휴일("5","7")만 Z2000050, 나머지 Z2000

**유팜 방식** (점수 기반):

```
가산 미적용:  금액 = (기준금액 × 기본점수).사사오입()
야간/공휴:   금액 = (기준금액 × Round(기본점수 × (1 + 공휴가산률), 2)).사사오입()
소아심야:    금액 = (기준금액 × Round(기본점수 × (1 + 소아야간가산률_기본), 2)).사사오입()
              ※ 2023.11.01 이후 기본조제기술료/복약지도료에만 소아야간가산률_기본 적용
토요가산:    기본금액 = (기준금액 × Round(기본점수, 2)).사사오입()
             토요금액 = (기준금액 × Round(기본점수 × 토요일가산, 2)).사사오입()
             합계 = 기본금액 + 토요금액  ← 분리 계산 후 합산
소아가산:    금액 = (기준금액 × (기본점수 + 소아가산점수)).사사오입()
```

**EDB 방식** (코드 조합 + 딕셔너리):

```
코드 = m_baseJojeSuga + 접미사(text + text2 + text3)
금액 = (int)mediWageM.GetPrice(코드)
토요: 별도 코드 = m_baseJojeSuga + text + "30"
```

**금액 산출 공식** (비즈팜):

```
ChkDosage = (in_Gb == "1") ? 1 : presDosage
Z2000_금액 = ChkDosage × insurance_amt
```

**의사코드**:

```python
def calc_Z2000(holidayGb, chkDate, presDate, presDosage):
    if chkDate:
        zCode = {
            "1": "Z2000010", "5": "Z2000050", "6": "Z2000600",
            "7": "Z2000650", "8": "Z2000610"
        }.get(holidayGb, "Z2000")
    else:
        zCode = "Z2000050" if holidayGb in ("5","7") else "Z2000"
    
    row = load_z_code(zCode, presDate)
    if row is None:
        return 0
    multiplier = 1 if row.in_Gb == "1" else presDosage
    return multiplier * row.insurance_amt
```

### 3-3. Z3000 복약지도료

**산정단위**: 방문당 1회

**Z코드 매핑 테이블** (`chkDate=True`):

| Holiday_gb | Z코드 | 의미 |
|------------|-------|------|
| `1`, `8` | Z3000010 | 야간 (6세+야간 포함) |
| `5`, `7` | Z3000050 | 공휴일 (6세+공휴일 포함) |
| 기타 | Z3000 | 기본 |

> `chkDate=False`: 항상 Z3000 (가산 코드 없음)

**특이사항**: Z2000과 달리 6세미만 단독 코드(Z3000600)가 없다. `Holiday_gb="6"`이면 기본 Z3000을 사용한다.

**금액 산출 공식**: Z2000과 동일 패턴 (`ChkDosage × insurance_amt`)

**의사코드**:

```python
def calc_Z3000(holidayGb, chkDate, presDate, presDosage):
    if chkDate:
        if holidayGb in ("1", "8"):
            zCode = "Z3000010"
        elif holidayGb in ("5", "7"):
            zCode = "Z3000050"
        else:
            zCode = "Z3000"
    else:
        zCode = "Z3000"
    
    row = load_z_code(zCode, presDate)
    if row is None:
        return 0
    multiplier = 1 if row.in_Gb == "1" else presDosage
    return multiplier * row.insurance_amt
```

### 3-4. Z4xxx 약품조제료 (핵심 -- 가장 복잡)

약품조제료는 **약품 유형 조합**에 따라 적용 항목과 코드가 달라진다.

#### (a) 외용만 처방 -- Z4120 (처방조제) / Z4220 (직접조제)

**산정단위**: 방문당 1회 (일수 무관, `ChkDosage=1` 고정)

**Z코드 매핑** (`chkDate=True`, 처방조제 Z4120 기준):

| Holiday_gb | Z코드 |
|------------|-------|
| `1`, `8` | Z4120010 |
| `5`, `7` | Z4120050 |
| 기타 | Z4120 |

**금액**: `1 × insurance_amt` (고정 1회)

#### (b) 내복 포함 처방 -- Z41xx/Z43xx (처방조제) / Z4200 (직접조제)

**처방조제**: 투약일수별 코드 결정 → 1회 적용 (`ChkDosage=1` 고정)

| 투약일수 | Z코드 (2008.01.01 이후) | Z코드 (2008.01.01 이전) |
|----------|------------------------|------------------------|
| 1~15일 | Z41{DD} (DD=2자리일수) | Z41{DD} |
| 16~20일 | Z4316 | Z4116 |
| 21~25일 | Z4321 | - |
| 26~30일 | Z4326 | Z4116 (16~30일) |
| 31~40일 | Z4331 | Z4117 (30일 초과) |
| 41~50일 | Z4341 | Z4117 |
| 51~60일 | Z4351 | Z4117 |
| 61~70일 | Z4361 | Z4117 |
| 71~80일 | Z4371 | Z4117 |
| 81~90일 | Z4381 | Z4117 |
| 91일+ | Z4391 | Z4117 |

야간/공휴일 접미사 (`chkDate=True`):

| Holiday_gb | 접미사 | 예시 (3일, 야간) |
|------------|--------|-----------------|
| `1`, `8` | `"010"` | Z4103010 |
| `5`, `7` | `"050"` | Z4103050 |
| 기타 | (없음) | Z4103 |

**금액**: `1 × insurance_amt` (고정 1회, 일수 곱하지 않음)

**직접조제**: Z4200 코드에 가산 접미사를 붙인 후 `금액 × 투약일수`

```
직접조제 내복조제료 = (기준금액 × 점수).사사오입() × 최대내복조제일수
```

이것이 처방조제와의 **핵심 차이점**이다. 유팜의 `직접조제료Logic.Get조제료Item()` 오버라이드에서 확인:

```csharp
if (type == 조제료CodeTypes.내복단독_직접)
    return num * (decimal)최대내복조제일수;  // 투약일수 배수!
```

**유팜 투약일수별 점수 분기** (처방조제):

```
1~15일  → Z4110_1 계열 (높은 점수, 예: P점수 4.82)
16일~   → Z4110_16 계열 (낮은 점수, 예: P점수 3.70)
```

유팜은 `Get기준금액(조제료CodeTypes.내복단독_처방, 최대내복조제일수)`에서:
- 16일 이상이면 코드명 + "3"으로 시작하는 항목 중 시작일/종료일 구간에 해당하는 것을 선택
- 15일 이하이면 코드명 + "1"로 시작하는 항목 중 `시작일 == 종료일 == 일수`인 것을 선택

#### (c) 내복+외용 동시 -- Z4121 (처방) / Z4221 (직접)

**산정단위**: 방문당 1회 추가 산정 (내복 조제료에 더해서)

**Z코드 매핑** (`chkDate=True`, 처방조제 기준):

| Holiday_gb | Z코드 |
|------------|-------|
| `1`, `7` | Z4121010 |
| `5`, `8` | Z4121050 |
| 기타 | Z4121 |

> **[주의 -- 버그 가능성]** Z4121의 Holiday_gb 매핑이 다른 Z코드와 **불일치**한다. 다른 코드는 `("1","8")`=야간, `("5","7")`=공휴일인데, Z4121은 `("1","7")`=010, `("5","8")`=050으로 매핑되어 있다. 비즈팜 원본 코드(라인 9556~9574)에서 확인. 유팜과 EDB에서는 이 패턴이 재현되지 않으므로 **비즈팜 고유의 버그 가능성**이 있다.

#### (d) 주사제만 처방 -- Z4130 (처방조제, 자가투여주사제만)

**조건**: 내복=0, 외용=0, 주사>0 AND 자가투여주사제=TRUE

**자가투여주사제가 아닌 경우**: 기본조제기술료/복약지도료/약국관리료 모두 0원, 약품조제료도 미산정 (약가만 발생)

**비즈팜**: 주사제만일 때 Z5000/Z5010만 적용하고 나머지 건너뜀

#### (e) 팩단위 -- Z4100 (처방) / Z4201 (직접)

모든 급여 내복약이 팩단위(팩수량>1)이면 팩단위 전용 코드를 사용한다.

```
[급여 팩단위 판정]
급여 내복약수 == (약품ID>0 AND 항목구분="0301" AND 팩수량>1 AND !향정
    AND (보험 OR 전액본인부담 OR 보훈전액본인부담))인 약품수
```

#### (f) 가루약 가산 -- Z4010 (처방) / Z4020 (직접)

**조건**: `내방일 >= 2023.11.01 AND 가루약 > 0`

```
가루약가산 점수 = Round(기본점수 × 가루약가산률, 2, AwayFromZero)
금액 = (기준금액 × 가루약가산 점수).사사오입()
```

**배타적 적용**: 가루약 가산 시 야간/공휴, 소아심야, 토요, 소아 가산 모두 미적용

**2023.11.01 이전**: `가루약=2`이면 별도 코드(Z4010/Z4020) 사용

**의사코드** (약품조제료 전체):

```python
def calc_Z4xxx(presDate, presDosage, rInChk, rOutChk, holidayGb, chkDate,
               is_direct, is_powder, 가루약가산률, ...):

    # 내복 포함
    if rInChk == "1":
        if is_direct:
            base_code = "Z4200"  # (or Z4201 for 팩단위)

            unit_price = load_price(zCode, presDate)
            total += unit_price * presDosage  # ★ 직접조제는 일수 곱셈
        else:
            base_code = determine_Z4_base_code(presDate, presDosage)
            zCode = base_code + 가산접미사(holidayGb, chkDate)
            total += load_and_calc(zCode, presDate, multiplier=1)
        
        # 가루약 가산 (별도)
        if is_powder and presDate >= "20231101":
            powder_score = round(base_score * 가루약가산률, 2)
            total += round(기준금액 * powder_score)
    
    # 내복+외용 동시
    if rInChk == "1" and rOutChk == "1":
        zCode = determine_Z4121_code(holidayGb, chkDate, is_direct)
        total += load_and_calc(zCode, presDate, multiplier=1)
    
    return total
```

### 3-5. Z5000 의약품관리료

**산정단위**: 방문당

**2단계 구조**:
1. **Z5000 기본료**: `ChkDosage × insurance_amt` (단, `insurance_amt=0`이면 가산하지 않음)
2. **Z5xxx 일수별 가산**: `1 × insurance_amt` (일수 곱하지 않음, 1회만 적용)

**일수별 가산 코드 결정** (2008.01.01 이후):

| 내복포함, 14~15일 | Z52{DD} |
| 내복포함, 16~20일 | Z5216 |
| 내복포함, 21~25일 | Z5221 |
| 내복포함, 26~30일 | Z5226 |
| 내복포함, 31~40일 | Z5231 |
| 내복포함, 41~50일 | Z5241 |
| 내복포함, 51~60일 | Z5251 |
| 내복포함, 61~70일 | Z5261 |
| 내복포함, 71~80일 | Z5271 |

> **핵심**: Z5000 기본료는 `ChkDosage×amt`, 일수별 가산은 `1×amt` (일수 곱하지 않음)

**마약 포함 시**: 향정약품이 있으면 Z5001(의약품관리료마약) 사용

**병팩 판정**: 모든 유효 약품이 (항목구분=0301, 팩수량>1, !향정)이면 Z5011, 투약일수 1일 고정

**처방조제 마약 분기 (유팜)**:

```
급여의약품관리료:
  향정약품 포함 (보험/전액본인부담/보훈전액본인부담) → Z5001
  미포함 → Z5

비급여의약품관리료:
  급여약 없음 AND 비급여마약적용 안함 → Z5
  급여약 없음 AND 비급여마약적용 AND 마약없음 → Z5
  급여약 없음 AND 비급여마약적용 AND 마약있음 → Z5001
  급여약 있음 → 0원
```

**의사코드**:

```python
def calc_Z5000(presDate, presDosage, rOutChk, rInChk, has_narcotic, all_pack):
    total = 0
    
    # 병팩 판정
    if all_pack:
        return load_price("Z5011", presDate)
    
    # 마약 판정
    base_code = "Z5001" if has_narcotic else "Z5000"
    
    # 1단계: 기본료
    row = load_z_code(base_code, presDate)
    if row and row.insurance_amt != 0:
        multiplier = 1 if row.in_Gb == "1" else presDosage
        total += multiplier * row.insurance_amt
    
    # 2단계: 일수별 가산
    if rOutChk == "1" and rInChk != "1":
        dosage_code = "Z5010"
    else:
        dosage_code = determine_Z5_dosage_code(presDate, presDosage)
    
    row2 = load_z_code(dosage_code, presDate)
    if row2:
        total += row2.insurance_amt  # 1회만 적용
    
    return total
```

## 5. 처방조제 vs 직접조제 차이

### 5-1. 점수 체계

| 항목 | 처방조제 | 직접조제 |
|------|---------|---------|
| 기준점수 | P점수 | D점수 |
| 공휴가산률 | P공휴가산률 | D공휴가산률 |
| 소아가산점수 | P소아가산점수 | D소아가산점수 |
| 내복 Z코드 | Z41xx / Z43xx | Z4200 |
| 내복팩단위 Z코드 | Z4100 | Z4201 |
| 외용 Z코드 | Z4120 | Z4220 |
| 내외용동시 Z코드 | Z4121 | Z4221 |
| 주사 Z코드 | Z4130 | (미배정) |
| 가루약 Z코드 | Z4010 | Z4020 |

### 5-2. 내복약 금액 계산 차이 (핵심)

```
처방조제:
  내복조제료 = (기준금액 × 점수).사사오입()  ← 일수 배수 미적용

직접조제:
  내복조제료 = (기준금액 × 점수).사사오입() × 최대내복조제일수  ← 일수 배수 적용
```

**예시** (투약일수 7일, 기준금액 90.4원, 점수 4.82):
- 처방: `round(90.4 × 4.82) = 436원`
- 직접: `436 × 7 = 3,052원`

### 5-3. 직접조제 전용: 일일 2회 내방

```
조건: 직접조제 AND 일일2회내방 = TRUE
효과: 기본조제기술료 = 0원, 약국관리료 = 0원
     (복약지도료, 의약품관리료는 유지)
```

### 5-4. 처방조제 전용: 달빛어린이 야간조제관리료

```
조건: 처방조제 AND 달빛어린이 = TRUE AND 합계기본조제료 > 0
효과: Z7001 야간조제관리료 추가 산정
```

## 6. 수가 조회 방식 비교

### 6-1. 비즈팜 (DB 직접조회)

```sql
SELECT in_Gb, insurance_amt
FROM PZCVALUE
WHERE drug_id = @zCode
  AND insurance_from <= @presDate
  AND insurance_to >= @presDate
```

- `in_Gb="1"` → 건당(1회), 그 외 → 일수별
- 금액 = `(in_Gb=="1" ? 1 : presDosage) × insurance_amt`

### 6-2. 유팜 (점수 × 기준금액)

```csharp
보험기준가 = 보험기준가List.Where(x => x.코드.Equals(type.GetEnumName())).SingleOrDefault();
기본점수 = Get기준점수(보험기준가);  // P점수 또는 D점수
금액 = (보험기준가.기준금액 × 최종점수).To원단위사사오입();
```

- 처방조제: `Get기준점수()` → `보험기준가.P점수`
- 직접조제: `Get기준점수()` → `보험기준가.D점수`
- 2024년 기준 `기준금액 = 90.4원/점`, 2026년 기준 `105.5원/점`

### 6-3. EDB (딕셔너리 단가 직접조회)

```csharp
int price = (int)mediWageM.GetPrice(sugaCode);
```

- `mediWageM`에 Z코드별 단가가 사전 로딩되어 있음
- 코드 = 기본코드 + 접미사(연령+시간+차등가산)
- 할증률(M_AddRat > 0) 적용 시: `price += (int)(price × M_AddRat / 100)`

### 7-2. 주요 불일치 사항

#### (1) Z4121 Holiday_gb 매핑 (비즈팜 버그 가능성)

```
비즈팜: Case "1","7" → Z4121010 / Case "5","8" → Z4121050
정상패턴: Case "1","8" → 야간 / Case "5","7" → 공휴일
```

유팜/EDB에서는 이 패턴이 재현되지 않음. **비즈팜 원본의 오타 또는 버그로 추정**. 구현 시 정상 패턴(`"1","8"`=야간, `"5","7"`=공휴일)을 따르되, 비즈팜과의 호환이 필요하면 별도 처리 필요.

#### (2) 토요가산 (비즈팜 미구현)

비즈팜 VB6 소스에는 토요가산 로직이 없다. 유팜(2013.10.1~ 단계적 도입)과 EDB(코드+"030")에는 있다.

```
유팜: 토요가산금액 = round(기본점수 × 토요일가산, 2)
     최종금액 = (기준금액 × 기본점수).사사오입 + (기준금액 × 토요가산금액).사사오입
EDB: 별도 코드 = baseCode + "030" → mediWageM.GetPrice()
```

#### (3) 비대면/가루약/코로나19 (비즈팜 미구현)

비즈팜 VB6는 2020년대에 추가된 비대면조제(ZC), 가루약가산, 코로나19 관련(ZH) 수가가 구현되어 있지 않다. 유팜과 EDB에서만 확인된다.

#### (4) 비급여 조제료 차액 계산

유팜과 EDB 모두 비급여 조제료를 **급여 초과분만** 청구하는 "차액 산정" 로직을 가진다:

```
비급여_내복조제료 = Max(비급여 산정값 - 급여 산정값, 0)
```

비즈팜도 `Pres_Bill(True)` 호출로 비보험 전용 조제료를 별도 계산하지만, 차액 로직의 세부 구현이 유팜/EDB보다 단순하다.

## 8. 주의사항

### 8-1. 가산 적용 우선순위 (상호 배타)

```
1순위: 가루약 가산 → 다른 모든 가산 배제 (소아 포함)
2순위: 야간/공휴 가산
3순위: 소아심야 가산
4순위: 토요 가산
추가: 소아 가산 (2~4순위와 중복 적용 가능, 단 가루약가산 시 미적용)
```

비대면 조제(ZC) 시 **모든 가산 미적용**.

### 8-2. 라운딩 규칙 (조제료 계산 내)

| 대상 | 규칙 |
|------|------|
| 점수 계산 중간값 | 소수 2자리 사사오입 `Math.Round(값, 2, AwayFromZero)` |
| 조제료 각 항목 금액 | 원단위 사사오입 `.To원단위사사오입()` |
| 조제료 단가 (공단 규정) | 점수 × 점수당단가 → 10원 미만 사사오입 |

### 8-3. 의약품관리료 투약일수 범위 매칭

유팜에서 의약품관리료(Z5)는 `시작일 <= 최대내복조제일수 && 종료일 >= 최대내복조제일수` 구간으로 매칭한다. 비즈팜/EDB도 동일하게 일수 구간별 코드를 사용한다.

### 8-4. 조제료 산정 여부 판정

```
미산정 조건 (하나라도 TRUE면 미산정):
  1) 약사본인조제여부 = TRUE
  2) 조제료산정안함 = TRUE AND 조제료미산정Type ∈ {전체(0), 보험(1)}

## 보충 A. 비급여 조제료 체계

> **출처**: `docs/약국_약제비_계산_분석.md` §7-1~7-4

### A-1. 비급여 조제료 환경설정 5종

약국 환경설정에서 비급여 약품에 대한 조제료 적용 범위를 선택한다.

| 비급여조제료Types | 값 | 설명 |
|------------------|---|------|
| 모두적용 | 0 | 비급여 내복/외용/주사 모두 산정 |
| 내복약 | 1 | 비급여 내복약만 산정 |
| 외용약 | 2 | 비급여 외용약만 산정 |
| 적용안함 | 3 | 비급여 조제료 미산정 |
| 주사제 | 4 | 비급여 주사제만 산정 |

### A-2. 추가 환경설정 옵션 3종

| 설정 | 기능 |
|------|------|
| **비급여기본조제료미산정** | TRUE 시 비급여 기본조제기술료/복약지도료/약국관리료 미산정. 의약품관리료와 약품조제료만 남음 |
| **비급여마약류관리료적용** | TRUE 시 비급여 마약 의약품관리료(Z5001) 산정. 2019.01.01 이후 기본값은 미산정이므로 이 옵션이 있어야 산정됨 |
| **비급여만조제료포함** | TRUE 시 비급여 약품만 있어도 조제료 산정. FALSE면 급여 약품이 최소 1개 이상 있어야 비급여 조제료도 산정 |

### A-3. 비급여 약품 존재 기본 조건

비급여 조제료 계산에 진입하려면 비급여 약품(내복+외용+주사)이 **1개 이상** 존재해야 한다. 비급여 약품이 없으면 비급여 조제료 전체가 미산정된다.

### A-4. 비급여 조제료 차액 산정 로직

비급여 조제료는 급여 조제료를 초과하는 부분만 청구한다. **차액 계산 대상 항목은 3개로 한정**된다:

```
차액계산 대상:
  비급여_내복조제료       = Max(비급여산정값 - 급여산정값, 0)
  비급여_내복조제료_가루약 = Max(비급여산정값 - 급여산정값, 0)
  비급여_내외용동시조제료  = Max(비급여산정값 - 급여산정값, 0)

차액계산 미대상 (비급여 산정값 그대로 사용):
  비급여_외용조제료       = 비급여산정값
  비급여_주사조제료       = 비급여산정값
```

> **이유**: 내복조제료와 내외용동시조제료는 급여/비급여 약품이 같은 처방에 섞일 때 중복 산정 방지가 필요하다. 외용조제료와 주사조제료는 급여와 비급여가 별도 산정되므로 차액 계산이 불필요하다.

### A-5. 비급여 마약 의약품관리료

```
비급여의약품관리료 결정:
  급여약 없음 AND 비급여마약류관리료적용 안함 → Z5 (일반)
  급여약 없음 AND 비급여마약류관리료적용 AND 마약없음 → Z5 (일반)
  급여약 없음 AND 비급여마약류관리료적용 AND 마약있음 → Z5001 (마약)
  급여약 있음 → 0원 (급여에서 이미 산정됨)

비급여 마약관리료 차액 = Z5001금액 - Z5금액 (양수만, 음수면 0원)
```

> 2019.01.01 이후 비급여 마약류 의약품관리료는 기본 미산정이다. 환경설정 `비급여마약류관리료적용=TRUE`일 때만 산정한다.

### A-6. 비급여 팩단위 판정

비급여에서는 급여와 다른 팩단위 판정 조건을 사용한다:

```
[비급여 팩단위 판정]
조건: 비급여 내복약수 == (약품ID>0 AND 항목구분="0301" AND 팩수량>1
      AND !보험 AND !전액본인부담 AND !보훈전액본인부담 AND !조제료산정여부)인 약품수

팩단위 코드: Z4100(처방) / Z4201(직접)
일반 코드: Z4(처방) / Z4200(직접)
```

> **급여와의 차이**: 비급여 팩단위 판정에는 `!조제료산정여부` 조건이 추가로 확인된다.

  복약지도료     = 0원

  비대면조제관리료 = 0원
```

> **핵심 차이**: 급여에서는 의약품관리료가 유지되지만, 비급여에서는 의약품관리료도 0원 처리된다.

### B-3. 보훈 1일 2회 이상 내방 특례

보훈_국비100, 보훈_감면100, 보훈_고엽제에 적용된다 (처방조제만).

```
[진입 조건]
  당일 2회 이상 내방 (매출서비스에서 조회)
  AND (내복조제일수 >= 1 OR 외용조제일수 == 1 OR 주사조제일수 == 1)

[처리 1: 모든 기본료 0원]
  기본조제기술료       = 0원
  복약지도료           = 0원
  약국관리료           = 0원
  의약품관리료         = 0원
  야간조제관리료       = 0원
  토요가산금액         = 0원
  코로나19투약안전관리료   = 0원
  코로나19대면투약관리료   = 0원

[처리 2: 내복 조제료 차감] (내복조제일수 >= 1일 때)
  ① 팩단위 재판정:
     현재 처방의 급여 내복약이 모두 팩단위인지 다시 확인
     (보훈전액본인부담 포함, 향정 제외)
  ② 재계산:
     Get조제료Item(재판정된코드, 보훈1일2회내방_내복조제일수)
  ③ 차감:
     최종 내복조제료 = 기존 내복조제료 - 재계산값
     (음수면 0원 처리)

[처리 3: 외용/주사 조제료 제거]
  외용조제일수 == 1 → 외용조제료 = 0원, 외용조제료_청구용 = 0원
  주사조제일수 == 1 → 주사조제료 = 0원
```

> **직접조제 1일2회 내방과의 차이**: 직접조제(§5-3)는 기본조제기술료/약국관리료만 제거하지만, 보훈 특례는 모든 기본료를 제거하고 약품조제료까지 재계산한다. 처리 범위가 훨씬 넓다.

### B-4. 보훈 클래스 상속 구조 (유팜)

```
처방조제료Logic (기본)
  └── 보훈국비100 (1일2회 내방 특례 구현)
      ├── 보훈감면100 (동일 로직 상속)
      └── 보훈고엽제 (동일 로직 상속)
```

보훈_감면30/50/60/90은 기본조제료 0원 처리만 수행하며, 1일2회 내방 특례는 적용하지 않는다.

## 보충 C. 조제료 미산정 조건

> **출처**: `docs/약국_약제비_계산_분석.md` §13-1~13-3

### C-1. 급여 조제료 미산정 조건

순서대로 평가하며, 하나라도 TRUE면 미산정이다.

```
[조건 1] 약사본인조제여부 = TRUE → 미산정

[조건 2] 조제료산정안함 = TRUE AND 조제료미산정Type ∈ {전체(0), 보험(1)} → 미산정

[조건 3] 약품별 조건 평가 (복합 로직):
  모든 유효약품(약품ID > 0)에 대해:
    IF 조제료산정여부 = TRUE:
      IF 비급여만조제료포함 = FALSE → 미산정 대상
      ELSE IF (!보험 AND !전액본인부담 AND !보훈전액본인부담) → 미산정 대상
      ELSE → 산정 대상 (급여 약품 존재)
    ELSE:
      → 미산정 대상 아님 (조제료 필요)

  모든 약품이 미산정 대상이면 → 조제료 미산정
```

> `조제료산정여부` 필드: 비급여 약품 중 별도 산정이 필요한 약품에 TRUE가 설정된다.
>
> `비급여만조제료포함`: TRUE면 비급여만 있어도 조제료 산정, FALSE면 급여 약품이 있어야 산정.

### C-2. 비급여 조제료 미산정 조건

```
[조건 1] 약사본인조제여부 = TRUE → 미산정

[조건 2] 조제료산정안함 = TRUE AND 조제료미산정Type ∈ {전체(0), 비보험(2)} → 미산정

[조건 3] 모든 약품의 조제료산정여부 = FALSE → 미산정
```

### C-3. 조제료미산정Types 분류

| 값 | 의미 | 급여 영향 | 비급여 영향 |
|---|------|----------|------------|
| 전체(0) | 급여+비급여 모두 미산정 | 미산정 | 미산정 |
| 보험(1) | 급여만 미산정 | 미산정 | 산정 |
| 비보험(2) | 비급여만 미산정 | 산정 | 미산정 |

### C-4. 주사제만 처방 시 기본조제료 0원 처리 (재확인)

```
[급여]
내복 + 외용 == 0 AND 주사 > 0 AND !자가투여주사제:
  → 기본조제기술료 = 0, 복약지도료 = 0, 약국관리료 = 0

[비급여]
급여약품 == 0 AND 비급여 내복외용 == 0 AND 비급여 주사 > 0 AND !자가투여주사제:
  → 기본조제기술료 = 0, 복약지도료 = 0, 약국관리료 = 0

[전체 약품수 = 0]
급여 + 비급여 약품수 == 0:
  → 기본조제기술료 = 0, 복약지도료 = 0, 약국관리료 = 0, 의약품관리료 = 0
```

### C-5. 조제료 산정 여부 판정 흐름도 (전체)

```
[진입]
├── Check급여조제료산정()
│   ├── 약사본인조제? → YES → 급여 미산정
│   ├── 조제료산정안함 AND (전체 OR 보험)? → YES → 급여 미산정
│   ├── 약품별 조건3 평가 → 모두 미산정 대상? → YES → 급여 미산정
│   └── PASS → 급여 조제료 산정 진행
│
└── Check비급여조제료산정()
    ├── 약사본인조제? → YES → 비급여 미산정
    ├── 조제료산정안함 AND (전체 OR 비보험)? → YES → 비급여 미산정
    ├── 모든 약품 조제료산정여부=FALSE? → YES → 비급여 미산정
    └── PASS → 비급여 조제료 산정 진행
```

> 급여/비급여 조제료 산정 여부는 독립적으로 판정된다. 급여가 미산정이어도 비급여는 산정될 수 있고, 그 반대도 가능하다.
