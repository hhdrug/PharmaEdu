# CH10. 약제비 계산 파이프라인

> **목적**: 다른 AI가 이 문서만 보고 약제비 계산 엔진의 전체 실행 순서를 구현할 수 있도록 정리한다.
> **근거 소스**: 비즈팜 VB6(`Insurance_Calculate`), 유팜 C#(`Base총약제비Logic.CalculateAll`), EDB C#(`PrsBillCalcM.Execute`), 공단 원문PDF 구현명세, `docs/약국_약제비_계산_로직.md`, `docs/약국_약제비_계산_분석.md`
> **최종 갱신**: 2026-04-09
> **주의**: 본문의 PDF 페이지 참조(p.147 등)는 **2024판 기준**. 2025판 기준 페이지는 각 CH 부록(CH01 부록, CH06 §10-1 등)에서 갱신 완료.

---

## 1. 개요 --- 전체 계산 엔진의 입력 -> 출력 흐름

```
┌─────────────────────────────────────────────────────────────────────┐
│                        약제비 계산 엔진                              │
│                                                                     │
│  [입력]                                                             │
│  ├─ 환자 정보 (나이, 보험유형, 공상등구분, 특정기호)                │
│  ├─ 처방 정보 (처방조제/직접조제, 발급기관종별, CT002)              │
│  ├─ 약품 목록 (코드, 단가, 투약량, 횟수, 일수, 본인부담률구분코드)  │
│  ├─ 시간/가산 조건 (야간, 공휴, 소아, 토요, 가루약)                 │
│  └─ 마스터 데이터 (수가표, 상한금액, 보험료율, 산정특례)            │
│                                                                     │
│  [처리] Step 1~10 파이프라인 (아래 상세)                             │
│                                                                     │
│  [출력]                                                             │
│  ├─ 요양급여비용총액1 (01+02항, 10원미만 절사)                      │
│  ├─ 요양급여비용총액2 (총액1 + U항, 10원미만 절사)                  │
│  ├─ 본인일부부담금 (100원미만 절사, 의료급여는 10원미만 절사)       │
│  ├─ 청구액 (총액1 - 본인부담)                                      │
│  ├─ 100/100 본인부담금총액 (U항, 10원미만 절사)                     │
│  ├─ 100/100미만 총액/본인부담/청구액 (A/B/D/E항 독립계산)          │
│  ├─ 보훈청구액 (해당 시)                                           │
│  ├─ 지원금 (희귀/긴급복지 해당 시)                                 │
│  ├─ 비급여금액 (W항)                                               │
│  └─ 조제투약내역 줄 목록 + 특정내역                                │
└─────────────────────────────────────────────────────────────────────┘
```

**핵심 불변 원칙:**
- `요양급여비용총액1`에는 01항(약가) + 02항(조제료)만 포함한다. A/B/D/E/U/V/W항은 절대 포함하지 않는다.
- 장려금(사용장려금, 대체조제장려금)은 총액1에 포함되지만, 본인부담 계산기준에서는 제외한다.
- 선별급여(A/B/D/E항)는 총액1과 완전히 독립된 별도 계산으로 관리한다.

## 2. 통합 파이프라인 (4소스 합의)

```
[Step 1] 입력 검증 + 초기화
    |
[Step 2] 마스터 데이터 로드 (수가/보험료율/산정특례)
    |
[Step 3] 약품 순회 -> 금액계산 + 분류 (→ CH01)
    |
[Step 4] 투약일수/보험용량 확정
    |
[Step 5] 시간/가산 조건 판정 (→ CH04)
    |
[Step 6] 수가 산정 (→ CH02/CH03)
    |
[Step 7] 보험료율 적용 + 본인부담금 (→ CH05)
    |
[Step 8] 3자배분 (→ CH06)
    |
[Step 9] 후처리 (특수케이스 → CH08)
    |
[Step 10] 결과 조립 + 출력
```

### Step 1: 입력 검증 + 초기화

**4소스 대응:**
| 소스 | 함수/메서드 | 설명 |
|------|------------|------|
| 비즈팜 | `Clear_Amt()` (L8945) | 모든 금액 변수, UI패널 초기화 |
| 유팜 | `CalculateAll()` 선두 — `new CalcResultsModel()` | 결과 모델 생성, 플래그 복사 |
| EDB | `Execute()` 단계1 — `m_prsBillM.ValueClear()`, `InitValue()` | 배열/변수 0초기화, 유효성검증 |
| 공단 | 구현명세 18단계 중 "1. 명세서 유형 결정" | H024/H124/H025/H125 분기 |

**처리 내용:**

```
1-1. 결과 저장소 초기화
     - 모든 금액 변수 = 0
     - 플래그 초기화: 내복약존재, 외용약존재, 주사약존재 = false
     - 항별 합계 배열 초기화: 01항, 02항, A~E항, U항, V항, W항 = 0

1-2. 입력 유효성 검증
     - 약품 목록이 비어있으면 → 조기종료 (§6 참조)
     - 보험유형 유효값 확인 (건강보험/의료급여/자보/산재/보훈 등)
     - 처방조제/직접조제 구분 확정
     - 서식번호 결정:
       처방조제 + 건강보험 → H024
       처방조제 + 의료급여 → H124
       직접조제 + 건강보험 → H025
       직접조제 + 의료급여 → H125

1-3. 환자 상태 플래그 설정
     - 야간여부, 공휴여부, 소아여부, 소아심야여부, 토요가산여부
     - 가루약여부, 비대면Type, 달빛어린이여부
     - 보훈코드 유무, 공상등구분 코드
```

### Step 2: 마스터 데이터 로드 (수가/보험료율/산정특례)

**4소스 대응:**
| 소스 | 함수/메서드 | 설명 |
|------|------------|------|
| 비즈팜 | `Pres_Bill()` L9220~9248 — PZCVALUE 테이블 조회 | ZBaseAmt, ZBaseold, ZBase 로드 |
| 유팜 | `Load보험기준가(적용일)` | 보험기준가 목록 + 조제료Logic.보험기준가List 설정 |
| EDB | `Execute()` 단계2 — `mediWageM = _repository.SelectMediWage(m_dosDate)` | 수가 마스터 로드 |
| 공단 | "4. 필수 코드/마스터" | Z코드 점수/단가, 약품별 상한금액 참조 |

**처리 내용:**

```
2-1. 수가 마스터 로드 (DueDate 기반)
     - 조제일자(dispense_date) 기준으로 해당 시점의 수가표 조회
     - 점수당 단가 로드 (예: 2026년 105.5원)
     - Z코드별 점수표 로드 (Z1000~Z7001)
     - 로드 실패 시 → 오류 반환 (계산 불가)

2-2. 보험료율 로드
     - 본인부담률 기준값: 30%(일반), 50%(V252), 40%(V352) 등
     - 65세 이상 정액 기준금액: 10,000원(정액경계), 12,000원(20%경계)
     - 직접조제 정액 기준금액: 4,000원(정률전환 경계)

2-3. 약품 마스터 참조
     - 약품코드별 상한금액
     - 퇴장방지의약품 여부
     - 보험등재 여부 (급여/비급여 판정)
     - 팩수량, 향정 여부, 성분 분류

2-4. 산정특례 마스터 참조
     - CT002 코드에 대응하는 산정특례 부담률
     - 산정특례 제4조(중증 5%), 제5조(희귀 10%), 제5조의2(결핵 0%), 제6조(V코드)
```

**출력 변수:**

| 변수 | 설명 |
|------|------|
| `point_unit_price` | 점수당 단가 (원) |
| `z_code_scores{}` | Z코드별 점수 맵 |
| `drug_ceiling_prices{}` | 약품코드별 상한금액 맵 |
| `copay_rates{}` | 보험유형/조건별 본인부담률 |

### Step 3: 약품 순회 -> 금액계산 + 분류 (→ CH01)

**4소스 대응:**
| 소스 | 함수/메서드 | 설명 |
|------|------------|------|
| 비즈팜 | `Check_Drug()` (L8642~8913) | sprDrug 전체 순회, 금액/분류 합산 |
| 유팜 | `약가Logic.Calculate급여약가()` 등 8종 호출 | 급여/비급여/전액본인부담/백대백미만A~E/보훈 약가 |
| EDB | `Execute()` 단계3 (L822~1244) | foreach DrugItem, PD_INSUPAY/PD_TAKE별 분류 |
| 공단 | "6. 줄 단위 산정 규칙" | 단가×투약량×횟수×일수, 원미만 4사5입 |

**처리 내용:**

```
FOR EACH drug IN drug_list:

    3-1. 약품코드 유효성 확인
         - 빈 코드 → skip
         - PD_EXTYPE 1,9 → skip (제외 대상)

    3-2. 단가 결정
         IF 실구입가 <= 상한금액:
             단가 = 실구입가
         ELSE:
             단가 = 상한금액
         IF 단가 < 1: 단가 = 1
         단가 = round(단가)  // 원미만 4사5입

    3-3. 금액 계산
         금액 = 단가 × 1회투약량 × 1일투약횟수 × 총투약일수
         금액 = round(금액)  // 원미만 4사5입

    3-4. 항 분류 (본인부담률구분코드 기준)
         (빈값) → 01항 (일반 급여)
         A → A항 (50% 선별급여)
         B → B항 (80%)
         D → D항 (30%)
         E → E항 (90%)
         U → U항 (100/100 본인부담)
         V → V항 (보훈 100/100)
         W → W항 (비급여)

    3-5. 목 분류 (약품분류 기준)
         01 → 01목 (내복약) → 내복약존재 = true
         02 → 02목 (외용)   → 외용약존재 = true
         03 → 03목 (주사)   → 주사약존재 = true

    3-6. 항별 합계 누적
         항_합계[해당항][해당목] += 금액

    3-7. 투약일수 추적 (내복/외용/주사별 최대값)
         급여_내복_최대일수 = MAX(급여_내복_최대일수, drug.총투약일수)
         급여_외용_최대일수 = MAX(급여_외용_최대일수, drug.총투약일수)
         비급여_내복_최대일수 = MAX(비급여_내복_최대일수, drug.총투약일수)

    3-8. 특수 약품 플래그
         - 퇴장방지의약품 → 사용장려금 대상 마킹
         - 대체조제(조제구분=4) → 장려금 계산 대상 마킹
         - 코로나19 치료제 → 급여 취급 강제
         - 팩단위 약품 카운트
         - 가루약(산제) 카운트 (ATB/ACH/AGN)

END FOR
```

**출력 변수:**

| 변수 | 설명 |
|------|------|
| `section_totals{항}{목}` | 항별/목별 금액 합계 |
| `has_internal` | 내복약 존재 여부 |
| `has_external` | 외용약 존재 여부 |
| `has_injection` | 주사약 존재 여부 |
| `max_internal_days` | 급여 내복약 최대 투약일수 |
| `max_external_days` | 급여 외용약 최대 투약일수 |
| `incentive_total` | 장려금 합계 (사용장려금 + 대체조제장려금) |
| `is_all_pack` | 모든 약품이 팩단위인지 |
| `powder_count` | 가루약 약품 수 |

### Step 4: 투약일수/보험용량 확정

**4소스 대응:**
| 소스 | 함수/메서드 | 설명 |
|------|------------|------|
| 비즈팜 | `Check_Drug()` 내 Dosage_cnt/TDosage_cnt 결정 | 최대 투약일수 갱신 |
| 유팜 | 약가Logic 내부에서 조제매출.투약일수 참조 | 조제일수인정 분기 |
| EDB | `Step4_CalcDrugDayAndInsuDose()` (L1246~1402) | num/num2(내복/외용 최대일수) 확정 |
| 공단 | "7-1. 투약일수" | 실투약일수 기준, 점안액/파스 특수 처리 |

**처리 내용:**

```
4-1. 급여 투약일수 확정
     - 내복약 조제일수 = 급여_내복_최대일수
       (단, 팩단위: 일수 = 1)
       (단, 병단위(ML/병): 일수 = 1)
     - 외용약 조제일수 = 급여_외용_최대일수 또는 1 (점안액 등)
     - 주사약 조제일수 = 급여_주사_최대일수

4-2. 비급여 투약일수 확정
     - 비급여_내복_최대일수 (비보험 조제료 산정용)
     - 비급여 팩단위/병단위 → 1일 고정

4-3. 가루약 조제일수 확정
     - 가루약=TRUE인 급여 내복약의 최대 투약일수

4-4. 특수 약품 투약일수 보정
     - 648903860 약품: MAX 5일 제한 (EDB 로컬 룰, 공식 고시 미확인 — CH01 §7-2 참조)
     - 포장단위(PD_PACK) 보정: 투약일수 ÷ 포장단위

4-5. 조기종료 판정 (약품 없음)
     - 유효 약품 수 = 0 → return (빈 결과)
     - 주사약만 존재 AND 자가투여=false → 조제료 미산정
```

**출력 변수:**

| 변수 | 설명 |
|------|------|
| `dispense_days_internal` | 내복약 조제일수 (확정) |
| `dispense_days_external` | 외용약 조제일수 (확정) |
| `dispense_days_powder` | 가루약 조제일수 (확정) |
| `injection_only_no_self` | 주사약단독+비자가투여 (조제료 미산정) |

### Step 5: 시간/가산 조건 판정 (→ CH04)

**4소스 대응:**
| 소스 | 함수/메서드 | 설명 |
|------|------------|------|
| 비즈팜 | `Check_Holiday()` (L8535~8640) | Holiday_gb 코드 결정 (N/1/5/6/7/8) |

```

     - 의료급여 + 특정 조건 → 조제료 = 0 리셋 후 재계산

9-7. 차등수가 처리 (심사청구서 수준)

     - 개별 명세서의 총액/본인부담은 변경 안 함

9-8. 본인부담상한액 초과 처리

## 3. 시점관리 (Temporal Logic)

### 3-1. 과거 처방 재계산 시 당시 수가/로직 소급

```
원칙: 계산에 사용하는 모든 수가/요율/로직은 "조제일자(dispense_date)" 기준의 것을 사용한다.
     2026년에 2025년 처방을 재계산할 때는 2025년 수가표와 점수당 단가를 적용해야 한다.

EDB 구현:
  - mediWageM = _repository.SelectMediWage(m_dosDate)
  → 조제일자 기준으로 수가 마스터 조회

  - 날짜별 수가코드 조정:
    IF dosDate >= 20230101: m_DrugSafeSuga_A = "ZH003"
    IF dosDate >= 20230101 AND 임신부: m_DrugSafeSuga_A = "ZH004"
    IF dosDate >= 20231101: m_PowderSuga = "" (가루약 별도 코드 폐지)

비즈팜 구현:
  - PZCVALUE 테이블에서 mskPresDate(처방일) 기준 조회

유팜 구현:
  - Load보험기준가(적용일) → 적용일 = 조제일자
  - 조제료Logic.내방일 = 적용일 (시점 동기화)
```

### 3-2. 수가 버전 관리 (DueDate 기반 MAX<=처방일)

```
수가 마스터 테이블 설계:
  | PK | 수가코드 | 적용시작일(DueDate) | 점수 | 단가 |

조회 쿼리:
  SELECT * FROM 수가마스터
  WHERE 수가코드 = @code
    AND 적용시작일 = (
      SELECT MAX(적용시작일) FROM 수가마스터
      WHERE 수가코드 = @code AND 적용시작일 <= @dispense_date
    )

이 패턴은 모든 마스터에 공통 적용:
  - 점수당 단가: 변동 시 새 행 추가 (예: 2026-01-01부터 105.5원)
  - Z코드 점수: 고시 변경 시 새 행 추가
  - 약품 상한금액: 고시 변경 시 새 행 추가
  - 본인부담 기준금액: 정책 변경 시 새 행 추가

EDB 구현 예시:
  - 2018.01.01 이전: 병원코드별 hg_Grade 보정 (하드코딩)
  - 보훈병원 6곳: 하드코딩된 기관코드로 판별

의료급여 정책 유보 관리:
  - 의료급여 본인부담 정률제 전환이 유보(2026-03 현재)
  → 외부 설정값으로 관리하여 정책 변경 시 코드 수정 없이 대응
```

처리: return false / 오류 반환

EDB: mediWageM == null → return false
유팜: _보험기준가 = null → 이후 조제료 계산 시 0원 처리
```

### 6-4. 의료급여 특수 리셋

```
조건: 보험유형=의료급여 + 특정 조건
시점: Step 7 이후 (비즈팜 STEP 4-1)
처리: Pres_Amt=0, Z1000~Z5000=0 리셋

비즈팜: cboBoHum5.Text="의료급여" → 조제료 전부 0
유팜:   의료급여 Logic에서 자체 처리
```

### 6-5. 급여제한/비보험 단독

```
조건: 급여제한여부=true OR (비보험 AND 보훈아님)
시점: Step 8 (Check청구액)
처리: 본인부담금 = 총약제비 전액, 청구액 = 0

유팜: Check청구액() — 급여제한이면 전액 본인부담
비즈팜: chkInsur=False → SelfRate() 직접 호출, 청구액 없음
```

## 8. 주의사항

### 8-1. 절대 혼합하지 말 것

```
요양급여비용총액1에는 01항 + 02항만 포함한다.
A/B/D/E항 → 별도의 "100/100미만" 독립 계산
U항       → 별도의 "100/100본인부담금총액" 독립 계산
V/W항     → 별도의 보훈/비급여 독립 관리
```

이 세 갈래를 한 합계로 섞는 것은 모든 소스에서 공통적으로 금지하고 있는 가장 중대한 오류이다.

### 8-2. 장려금 이중성

장려금(사용장려금 + 대체조제장려금)은:
- 01항의 해당 목에 **포함**된다 → 요양급여비용총액1에 포함
- 본인부담 계산기준에서는 **제외**된다 → 본인부담_기준액 = 총액1 - 장려금합계
- 전액 **청구액(공단부담)**에 포함된다

### 8-3. 절사 함수 분기

| 보험유형 | 본인부담 절사 | 총액 절사 |
|----------|-------------|----------|
| 건강보험 | trunc100 (100원미만 절사) | trunc10 (10원미만 절사) |
| 의료급여 | trunc10 (10원미만 절사) | trunc10 (10원미만 절사) |
| 차상위(V252 3%) | trunc100 (건보) / trunc10 (의료급여) | trunc10 |

### 8-4. 산정특례 + 보훈 동시 적용

산정특례(5%/10%)와 보훈감면(30%~90%)은 동시에 적용된다.
보훈감면 후 남은 금액에 산정특례 부담률을 곱한다 (30% 대신).

### 8-5. 65세 정액 + 보훈 조합

보훈감면 환자가 65세 이상이더라도 65세 정액 1,000원 규칙은 적용되지 않는다.
보훈감면 분기가 일반 분기보다 먼저 처리되므로 자연스럽게 반영된다.

### 8-6. CT002 판단 주체

CT002(V252/V352/V452)는 의료기관이 처방전에 인쇄하는 것이다.
약국은 처방전에 인쇄된 값을 그대로 입력하며, 자체적으로 판단/생성하지 않는다.
CT002가 비어있으면 산정특례 제6조 미해당으로 처리한다.

### 8-7. 의료급여 본인부담 외부 설정화

의료급여 본인부담은 정책 변동 가능성이 높다 (정률제 전환 논의 유보 중).
1종/2종/행려별 금액을 외부 설정값(별도 테이블)에서 읽어오도록 설계하고,
하드코딩하지 않아야 한다.

### 8-8. 재계산 트리거 인식

비즈팜 분석에서 Insurance_Calculate()가 46곳에서 호출되는 것으로 확인된다.
보험구분 변경(11곳), 약품 추가/삭제(8곳), 투약일수 변경(3곳), 처방전 로드(8곳) 등
다양한 사용자 이벤트에서 전체 파이프라인이 재실행된다.
구현 시 이 재계산 트리거를 적절히 설계해야 한다.

### 8-9. 비보험 조제료 별도 산정

비즈팜은 보험 조제료와 비보험 조제료를 Pres_Bill(False)와 Pres_Bill(True)로 별도 산정한다.
유팜도 Calculate급여조제료()와 Calculate비급여조제료()를 분리한다.
비보험 약품에 대한 조제료는 독립적으로 산정하여 비보험금액에 합산해야 한다.

### 8-10. 약국에는 종별가산율이 없다

의과에서는 진료행위에 종별가산율(의원 15%, 병원 20% 등)을 곱하지만,
약국의 약가와 조제료에는 종별가산율이 적용되지 않는다.
따라서 약국 총액1 = 01항(약가) + 02항(조제료)이며, 별도의 가산율 적용 단계가 없다.
