## 1. 개요 — 3자배분이란

약국에서 발생한 약제비(요양급여비용총액1)를 **3개 주체**에게 배분하는 과정이다.

| 주체 | 변수명 | 의미 |
|------|--------|------|
| **환자** | `UserPrice` (= 본인일부부담금) | 환자가 약국 창구에서 직접 납부하는 금액 |
| **공단** | `InsuPrice` (= 청구액) | 건강보험공단(또는 의료급여기관)에 청구하는 금액 |
| **보훈** | `MpvaPrice` (= 보훈청구액) | 국가보훈처(보훈병원)에 청구하는 금액 |

### 항등식 (불변 조건)

```
요양급여비용총액1 = UserPrice + InsuPrice + MpvaPrice
```

보훈 환자가 아닌 경우 `MpvaPrice = 0`이므로:
```
요양급여비용총액1 = UserPrice + InsuPrice
```

### 추가 파생 변수

| 변수 | 의미 |
|------|------|
| `MpvaComm` | 보훈 비급여분(100%약품+조제관리료 등)에 대한 보훈부담분 |
| `PubPrice` | 공비(희귀질환/긴급복지/차상위 등 제3기관 지원금) |
| `RealPrice` | 실수납금 = UserPrice - PubPrice |
| `SumUser` | 최종 환자수납액 = RealPrice + 비급여약품 - MpvaComm + 100%약품_환자분 |
| `SumInsure` | 최종 공단청구액 (= InsuPrice, 약안전가산 등 보정 후) |
| `OverUserPrice` | 본인부담상한제 초과금 (환자에서 공단으로 전환) |

## 2. 기본 3자배분 공식

### 2-1. 일반 환자 (보훈 없음)

```
UserPrice = trunc100(총액1 × 본인부담률)     // 건강보험: 100원 절사
InsuPrice = 총액1 - UserPrice                // 잔액 = 공단청구액
MpvaPrice = 0
```

**소스 근거 (EDB InsuRateCalc2 원본 3555~3600):**
```csharp
// C타입(건강보험) 일반 정률
UserPrice = num - MpvaPrice;  // num = 총액1(급여)
UserPrice = Math.Truncate(UserPrice * (insuRate / 100m) / 100m) * 100m;
```

**소스 근거 (공단 문서 p.663):**
```
청구액 = 요양급여비용총액1 - 본인일부부담금
```

### 2-2. 보훈감면 환자 (보훈병원 처방전)

**공단 문서 p.575~576 원문 확인:**

```
보훈청구액 = trunc10(요양급여비용총액1 × 감면율)
본인부담_기준액 = 요양급여비용총액1 - 보훈청구액
본인일부부담금 = trunc100(본인부담_기준액 × 본인부담률)
청구액 = 요양급여비용총액1 - 보훈청구액 - 본인일부부담금
```

**감면율 표:**

| 공상등구분 | 감면율 | EDB bohunCode | 예시(총액 20,000원, 정률 30%) |
|-----------|--------|---------------|-------------------------------|
| 3 | 30% | M30 | 보훈 6,000 / 본인 4,200 / 공단 9,800 |
| 5 | 50% | M50 | 보훈 10,000 / 본인 3,000 / 공단 7,000 |
| 6 | 60% | M60 | 보훈 12,000 / 본인 2,400 / 공단 5,600 |
| J | 90% | M90 | 보훈 18,000 / 본인 600 / 공단 1,400 |
| 4/7 (국비) | 100% | M10 | 보훈 20,000 / 본인 0 / 공단 0 |

### 2-3. 보훈국비 100% (M10)

**유팜 보훈국비100.cs 확인:**
```csharp
// 보훈감면율 = 1m (100%)
보훈청구액 = 총약제비;  // 전액 보훈
본인부담금 = 0;
// 100%미만 약품(A/B/D/E)도 전부 보훈으로 전환:
백대백미만보훈청구액 = 백대백미만본인부담금 + 백대백미만청구액;
백대백미만본인부담금 = 0;
백대백미만청구액 = 0;
```

**EDB InsuRateCalc3 원본 3917~3920:**
```csharp
// M10 → InsuPrice에서 Under_Insu 차감, MpvaPrice에 가산
if (bohunCode == "M10")
{
    InsuPrice -= SumUserDrug_100_Under_Insu;
    MpvaPrice += SumUserDrug_100_Under_Insu;
}
```

## 3. 공비(PubPrice) 로직

공비는 환자 본인부담금 중 제3기관(시군구/복지기관 등)이 대신 부담하는 금액이다.

### 3-1. 공비 결정 규칙 (EDB InsuRateCalc3 원본 3863~3908)

```
PubPrice = 0  // 기본값

IF 희귀질환(isRare) AND 본인부담률 != 100%:
    PubPrice = UserPrice    // 환자 부담 전액을 공비로 전환

IF G타입(긴급복지):
    PubPrice = UserPrice    // 환자 부담 전액을 공비로 전환

IF C타입(건강보험) AND (특수공비 != "N" AND 특수공비 != "102") OR IND == "Y":
    PubPrice = UserPrice    // 환자 부담 전액을 공비로 전환
```

### 3-2. V246/V206 특수공비 (C타입 전용)

```
IF C타입 AND (질병코드 == "V246" OR 질병코드 == "V206"):
    IF 조제일자 < 20160701:
        PubPrice = trunc10(UserPrice × 50%)
    ELSE:
        // 2016.07.01 이후에는 이 분기 미적용 (다른 규칙 적용)
```

**EDB 원본 3896~3908:**
```csharp
if (insuCode[0] == 'C')
{
    switch (pm_Medi_Illness)
    {
        case "V246":
        case "V206":
            if (dosDate < 20160701m)
                PubPrice = Math.Truncate(UserPrice * 0.5m / 10m) * 10m;
            break;
    }
}
```

### 3-3. 공비 유형 정리표

| 조건 | PubPrice | 비고 |
|------|----------|------|
| 희귀질환 + 비전액 | = UserPrice | 환자부담 전액 공비 전환 |
| G타입 (긴급복지) | = UserPrice | 환자부담 전액 공비 전환 |
| C타입 + 특수공비(N/102 아닌) | = UserPrice | 차상위 등 |
| C타입 + IND=Y | = UserPrice | 산재 해당 |
| C타입 + V246/V206 + 2016.07.01 이전 | = trunc10(UserPrice * 50%) | 반액 공비 |
| D타입 302/101 | PubPrice 대신 Pub100Price | 4절 참조 |
| G타입 302 | Pub100Price = SumUser, SumUser = 0 | 4절 참조 |
| 해당없음 | 0 | |

### 3-4. F008 (긴급복지 전액면제)

```
IF 질병코드B == "F008":
    PubPrice += RealPrice + F008_100DrugCost
    RealPrice = 0
```

**EDB 원본 3945~3950:** F008은 본인부담 0원이면서 공비로 전액 전환한다.

## 4. 특수공비 (302/101/102) 재배분

특수공비 코드는 `M_Ps_Special_Pub`에 저장되며, 보험유형(C/D/G)별로 다르게 처리된다.

### 4-1. C타입 (건강보험) 특수공비

**EDB InsuRateCalc3 원본 4002~4028:**

| 특수공비 | 비급여미적용(NPay=N) | 비급여적용(NPay=Y) |
|---------|-------------------|--------------------|
| **302** | Pub100Price = SumInsuDrug_100<br>SumUser -= SumInsuDrug_100 | Pub100Price = SumUser<br>SumUser = 0 |
| **101** | Pub100Price = SumInsuDrug_100<br>SumUser -= SumInsuDrug_100 | (동일) |
| **기타** | SumUser -= SumInsuDrug_100<br>InsuPrice += SumInsuDrug_100<br>SumInsure += SumInsuDrug_100 | (동일) |
| **102** | (처리 없음 — break) | (처리 없음) |

**의사코드:**
```
IF C타입:
    SWITCH Special_Pub:
        CASE "302":
            IF NPay_Exp == "N":
                Pub100Price = SumInsuDrug_100
                SumUser -= SumInsuDrug_100
            ELSE:
                Pub100Price = SumUser
                SumUser = 0
        CASE "101":
            Pub100Price = SumInsuDrug_100
            SumUser -= SumInsuDrug_100
        CASE "102":
            // 미처리 (pass)
        DEFAULT:
            SumUser -= SumInsuDrug_100
            InsuPrice += SumInsuDrug_100
            SumInsure += SumInsuDrug_100
```

### 4-2. D타입 (의료급여) 특수공비

**EDB 원본 4030~4051:**

| 특수공비 | 비급여미적용(NPay=N) | 비급여적용(NPay=Y) |
|---------|-------------------|--------------------|
| **302** | Pub100Price = SumInsuDrug_100<br>SumUser -= SumInsuDrug_100 | Pub100Price = SumUser<br>SumUser = 0 |
| **101** | Pub100Price = SumInsuDrug_100<br>SumUser -= SumInsuDrug_100 | (동일) |

### 4-3. G타입 (긴급복지 등) 특수공비

**EDB 원본 4053~4062:**

| 특수공비 | 처리 |
|---------|------|
| **302** | Pub100Price = SumUser<br>SumUser = 0 |
| 기타 | (처리 없음) |

### 4-4. Pub100Price의 의미

`Pub100Price`는 100% 약품(U항/비급여 등) 중 공비로 전환되는 금액이다. 이 값은 `SumUser`에서 차감되어 환자가 내지 않게 된다.

## 5. 보훈 MpvaPrice/MpvaComm 산출

### 5-1. 용어 정리

| 변수 | 역할 |
|------|------|
| `MpvaPrice` | 보훈청구액 (급여분에 대한 보훈 부담) |
| `MpvaComm` | 보훈 비급여관리료분 (비급여약품 조제관리료 + 100%약품에 대한 보훈 부담) |
| `M_isMPV_Bill` | 위탁(보훈병원 처방전) 여부 |
| `M_isBohunHospigtal` | 보훈약국(보훈병원 내 약국) 여부 |
| `M_isSimSa` | 심사기관 여부 |
| `m_bohunRate` | 보훈감면율 (30/50/60/90/100) |
| `m_bohunCode` | 보훈코드 (M10/M20/M30/M50/M60/M61/M81/M82/M83/M90) |
| `num7` | M20/M61 이중감면률 (보훈감면 위에 추가 감면) |

### 5-2. MpvaPrice 산출 (EDB InsuRateCalc2 원본 3078~3101)

**조건:** `bohunRate > 0 AND bohunCode != "M20" AND bohunCode != "M61"`

#### (A) 위탁(MPV_Bill) + 급여총액==약가총액인 경우 → 역산 방식

```csharp
// 원본 3082~3085
IF MPV_Bill AND SumWage == num AND num2 == SumInsuDrug_100:
    temp = trunc10(num * ((100 - bohunRate) / 100))  // 비보훈분
    MpvaPrice = num - temp                            // 역산 = 보훈분
```

**역산의 의미:** `총액 - trunc10(총액 * 비보훈비율)` = 보훈분. 이렇게 해야 절사 오차가 환자에게 불리하지 않다.

#### (B) 위탁 + 일반 경우

```csharp
// 원본 3089
MpvaPrice = trunc10(num * (bohunRate / 100))  // 정산 방식
```

#### (C) 특수약품(648903860) 존재 시

```csharp
// 원본 3094~3096
num11 = trunc100(((num - num9) * (insuRate/100) + num10) * ((100 - bohunRate)/100))
MpvaPrice = num - trunc10(num * ((100 - bohunRate) / 100))
```

#### (D) 비위탁 일반 → 역산 방식

```csharp
// 원본 3100~3101
MpvaPrice = trunc10(num * ((100 - bohunRate) / 100))  // 비보훈분
MpvaPrice = num - MpvaPrice                            // 역산 = 보훈분
```

#### (E) D/C21/C31/C32 + M10이 아닌 경우 → MpvaPrice 강제 0

```csharp
// 원본 3103~3106
IF (D타입 OR C21/C31/C32) AND bohunCode != "M10":
    // 2019 이후 D+B014 제외
    IF NOT (dosDate >= 20190101 AND sbrdnType == "B014" AND D타입):
        MpvaPrice = 0
```

### 5-3. MpvaComm 산출

MpvaComm은 비급여약품(num3)과 조제관리료(SumWageComm), 100%이하약품(SumUserDrug_100_Under_User) 등에 대한 보훈 부담분이다.

#### D/C21/C31/C32 + M10인 경우 (원본 3107~3138)

```
IF 보훈_비급여_본인부담 아님:
    IF num3(비급여약품) == 0:
        IF 비보훈약국:
            MpvaComm = trunc10(100%이하약품_환자분 * bohunRate/100)
    ELSE IF 보훈약국:
        MpvaComm = trunc10((num3 + SumWageComm) * bohunRate/100)
    ELSE:
        MpvaComm = trunc10((num3 + SumWageComm + 100%이하약품_환자분) * bohunRate/100)
ELSE: // 보훈_비급여_본인부담
    MpvaComm = trunc10((SumWageComm + num4) * bohunRate/100)
```

#### 일반 보험유형 (D/C21/C31/C32 이외) — 원본 3144~3256

위탁/비위탁, 심사/비심사, 보훈약국 여부에 따라 세분화된다.

**핵심 패턴 — 비위탁(비MPV) + 비심사 + 비보훈약국:**
```csharp
// 원본 3243~3250
temp = trunc10((num3 + SumWageComm) * ((100 - bohunRate)/100))
IF bohunRate == 100:
    MpvaComm = num3 + SumWageComm + SumUserDrug_100_Under_User - temp
ELSE:
    MpvaComm = num3 + SumWageComm - temp  // 역산
```

### 5-4. M20 이중감면 (num7)

M20은 본인부담에서 한 번 더 감면하는 2차 감면이다.

**EDB InsuRateCalc2 원본 3616~3619 (C타입 일반):**
```csharp
IF bohunCode == "M20":
    MpvaPrice = UserPrice - trunc100(UserPrice * (num7/100))
    UserPrice -= MpvaPrice
    // MpvaPrice = 보훈이 추가 부담하는 금액
    // UserPrice = 환자 최종 부담
```

**D타입 M20 (원본 3773~3783):**
```csharp
IF bohunCode == "M20":
    IF dosDate >= 20180101:
        MpvaPrice = UserPrice - trunc10(UserPrice * (num7/100))   // 10원 절사
    ELSE:
        MpvaPrice = UserPrice - trunc100(UserPrice * (num7/100))  // 100원 절사
    UserPrice -= MpvaPrice
```

**G타입(default) M20 (원본 3807~3818):**
```csharp
IF bohunCode == "M20":
    MpvaPrice = UserPrice - trunc100(UserPrice * (num7/100))
    UserPrice -= MpvaPrice
    MpvaPrice = 0   // ★ G타입에서는 MpvaPrice를 0으로 리셋
    // 대신 MpvaComm으로 전환
    IF num3 == 0:
        MpvaComm = 0
    ELSE:
        MpvaComm = trunc100((num3 + SumWageComm) * (num7/100))
    // 최종:
    MpvaComm = num3 + SumWageComm - MpvaComm  // 역산
```

### 5-5. M61 고엽제 역산

M61은 고엽제 피해자 감면으로, 본인부담률과 보훈감면률을 **동시에** 곱한다.

**EDB 원본 3594~3596 (C타입):**
```csharp
IF bohunCode == "M61" AND num7 >= 0:
    UserPrice = trunc100(UserPrice * (insuRate/100) * (num7/100))
    // 예: 본인부담률 30% × 이중감면률 50% = 15%
```

**M61 + 특수약품(648903860) 없는 경우 (원본 3635~3643):**
```csharp
IF bohunCode == "M61" AND num9 == 0:
    MpvaPrice = trunc100(num * (insuRate/100)) - UserPrice
    // MpvaPrice는 본인부담률 적용 후 잔액에서 UserPrice를 뺀 것
```

## 7. 본인부담상한제 (OverUserPrice)

연간 본인부담금 누적액이 상한액을 초과하면 초과분을 공단이 부담한다.

### 7-1. EDB InsuRateCalc3 원본 4146~4169

```csharp
IF OverUserPriceYN == "Y":
    // 기존 누적 RealPrice 조회
    pb_RealPrice = GetPbRealPrice(환자코드, 조제일자)
    // 연간상한액 조회
    configValue = GetConfigValue("P", "P16", 연도)

    IF configValue > 0:
        pb_RealPrice += 이번_RealPrice   // 누적에 이번 건 추가
        초과분 = pb_RealPrice - configValue

        IF 초과분 > 0:
            IF 이번_RealPrice < 초과분:
                // 이번 건 전체가 초과
                OverUserPrice = RealPrice
                RealPrice = 0
            ELSE:
                // 이번 건 일부만 초과
                RealPrice -= 초과분
                OverUserPrice = 초과분

            SumUser -= OverUserPrice    // 환자수납에서 차감
```

### 7-2. 핵심 포인트

- `OverUserPrice`는 `SumUser`에서만 차감되고, `InsuPrice`에는 반영되지 않는다
- 연간상한액은 연도별로 설정값(P16)에서 조회한다
- 이 기능은 약국 옵션(`M_OVER_USER_PRICE_YN`)으로 활성화/비활성화 가능하다

### 10-1. 4소스 일치 사항

### 10-2. 소스별 차이점

#### (A) 비즈팜 50%감면+암환자 BUG

- **위치:** 비즈팜 006 라인 9732
- **현상:** `Insur_Self_Amt = Insur_Self_Amt = Int(...)` → 이중 대입으로 비교 결과(0/-1)가 저장됨
- **영향:** 50%감면+암환자+조합기호 조합에서 본인부담금이 0 또는 -1이 됨
- **다른 소스:** EDB/유팜에서는 이 BUG 없음
- **결론:** 비즈팜 고유 BUG — 구현 시 무시하고 정상 로직 적용

#### (B) 비즈팜 dead code (intZBaseAmt 비교)

- **위치:** 비즈팜 006 라인 9825
- **현상:** `intZBaseAmt <> intZBaseAmt`는 항상 False
- **영향:** Else 분기가 실행되지 않음
- **결론:** dead code — 구현 시 무시

#### (C) M20 G타입에서 MpvaPrice 리셋

- **EDB:** M20 + G타입 → `MpvaPrice = 0`으로 리셋 후 MpvaComm으로 처리 (원본 3809~3818)
- **유팜:** 명시적 리셋 코드 미확인 (Calculate보훈 내부에서 별도 처리 추정)
- **결론:** EDB 로직 기준으로 구현 (G타입에서 M20의 급여분 보훈부담은 MpvaPrice가 아닌 InsuPrice→MpvaPrice 전환으로 처리)

#### (D) D타입 M20 절사 단위 시기별 차이

- **EDB:** 2018.01.01 이전 100원 절사, 이후 10원 절사 (원본 3773~3783)
- **비즈팜/유팜:** 시기별 차이 명시 미확인
- **결론:** EDB 로직이 가장 세밀 — EDB 기준으로 구현

#### (E) 보훈감면 + 65세 정액 우선순위

- **docs 약국_약제비_계산_로직.md §7-4:** "보훈감면 환자가 65세 이상이고 약제비 1만원 미만이더라도, 65세 정액 1,000원 규칙은 적용되지 않는다."
- **유팜 보훈감면Base.cs (라인 31~39):** 65세 + 정액노인 금액 체크를 하되, `산정특례본인부담율사용==false`인 경우만. 보훈감면율로 `본인부담율 = 1 - 보훈감면율`을 설정.
- **EDB:** 65세 분기 후 보훈코드 체크로 자연스럽게 보훈 우선
- **결론:** 4소스 모두 보훈감면이 65세 정액보다 우선 — **일치**

### 10-3. 모순 식별 결과

**모순 없음.** 4소스 간 근본적인 산식 모순은 발견되지 않았다. 비즈팜의 2건의 BUG는 비즈팜 고유 문제이며 나머지 3소스와 공단 문서가 일치하므로, 정상 로직 기준으로 구현하면 된다.

### 11-2. 역산 패턴

EDB에서 MpvaPrice를 산출할 때 **역산**을 자주 사용한다:
```
MpvaPrice = 총액 - trunc10(총액 * (100-감면율)/100)
```

이 방식은 `trunc10(총액 * 감면율/100)`과 결과가 다를 수 있다(절사 방향 차이). EDB는 **비보훈분을 먼저 절사하고 나머지를 보훈분으로** 취하는 역산 방식을 사용한다. 이는 절사 오차가 환자에게 불리하지 않도록 하는 설계이다.

### 11-3. InsuPrice(공단청구액) 산출 방식

InsuPrice는 직접 계산하지 않고 **잔액**으로 구한다:

```
// 심사기관:
InsuPrice = num - MpvaPrice - UserPrice + num6

// 일반(비심사):
InsuPrice = num - MpvaPrice - UserPrice + num6 + SumUserDrug_100_Under_Insu
```

`num6`은 토요가산 보정액이다.
